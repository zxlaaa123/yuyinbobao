## 一、Critical 级别（可能导致崩溃/数据丢失/安全漏洞）

### C-1：`retryAudioFile` 函数未导出（前端运行时崩溃）
- **文件**：`frontend/src/api/audio.ts`（缺失）、`frontend/src/views/AudioView.vue`（调用方）
- **问题**：`AudioView.vue` 中 `handleRetry` 调用了 `retryAudioFile(audio.id)`，但 `audio.ts` 中没有导出此函数。点击"重新生成"按钮会直接报 `TypeError: retryAudioFile is not a function`。
- **触发场景**：用户在音频播报页面点击任意音频的"重新生成"按钮。
- **修复**：在 `audio.ts` 中添加 `export function retryAudioFile(id: number) { return request.post('/api/audio-files/${id}/retry') }`

### C-2：小米 TTS 使用了错误的 API 端点
- **文件**：`backend/app/services/tts_service.py`（`_xiaomi_synthesize`）
- **问题**：使用 `/chat/completions` + `api-key` 头 + `messages/audio` 参数，这是 OpenAI 聊天接口格式，不是 TTS 接口。小米 TTS 应使用专用 TTS 端点。
- **触发场景**：用户选择 xiaomi 作为 TTS Provider 时，所有音频生成都失败。
- **修复**：确认小米 TTS 真实 API 文档后修正；文档不明确前标记为未实现。

### C-3：`import-and-extract` 中异常路径下返回数据不可靠
- **文件**：`backend/app/api/routes/materials.py`
- **问题**：AI 提取成功但 `db.commit()` 失败时，`all_created` 中的 point 对象 ID 为 None，返回语句引用 `point.id` 会产生不可预期的结果。
- **触发场景**：磁盘满或约束冲突导致 commit 失败。
- **修复**：在 return 前检查 ID 是否为 None，做保护处理。

## 二、High 级别（可能导致功能异常/数据不一致）

### H-1：所有 ForeignKey 缺少 `ondelete` 约束
- **文件**：所有 `backend/app/models/*.py`
- **问题**：所有 ForeignKey 均未指定 `ondelete`。虽然路由层手动处理了部分级联删除，但 `StudySession`、`PracticeSession`、`PracticeSessionItem`、`Flashcard`、`AudioFile` 等表的关联记录会成为孤儿。
- **触发场景**：删除知识库/知识点后，相关会话记录、卡片记录成为孤立数据。
- **修复**：为所有 ForeignKey 添加 `ondelete="CASCADE"` 或 `ondelete="SET NULL"`。

### H-2：`create_ai_call_log_independent` 中会话泄漏
- **文件**：`backend/app/services/ai_log_service.py`
- **问题**：创建了新的 `SessionLocal()` 但没有 `finally: db.close()`。如果异常发生，会话永远不会关闭，导致连接泄漏。
- **触发场景**：AI 日志写入失败时，数据库连接泄漏。
- **修复**：添加 `finally: db.close()`。

### H-3：`_write_log` 异常被完全吞掉
- **文件**：`backend/app/services/ai_service.py`
- **问题**：`_write_log` 的 `except Exception: pass` 完全吞掉异常，日志写入失败时无任何记录。
- **触发场景**：数据库连接异常时，AI 操作成功但日志缺失，且无任何提示。
- **修复**：至少记录 `logger.warning`。

### H-4：SQLite 并发写入丢失更新
- **文件**：`backend/app/core/database.py` + `backend/app/api/routes/practice.py`
- **问题**：`submit_answer` 中先查询 `WrongQuestion`，再更新，最后 `commit`。并发提交同一题目的答案时，`wrong_count` 可能丢失更新。
- **触发场景**：快速连续提交同一道题目的答案。
- **修复**：使用原子更新 `UPDATE wrong_questions SET wrong_count = wrong_count + 1`。

### H-5：搜索竞态条件（前端）
- **文件**：`frontend/src/layouts/MainLayout.vue`
- **问题**：`runSearch` 没有取消机制，快速输入时后发出的请求可能先返回，覆盖正确结果。
- **触发场景**：用户快速输入搜索关键词。
- **修复**：使用请求序列号或 `AbortController` 丢弃过期响应。

### H-6：练习会话保存竞态条件（前端）
- **文件**：`frontend/src/views/PracticeView.vue`
- **问题**：`finishCurrentSession` 中 `createPracticeSession` 和 `finishStudySession` 没有错误隔离，一个失败会影响另一个。
- **触发场景**：网络不稳定时，学习会话保存失败导致练习会话也不保存。
- **修复**：两个保存操作独立，分别 try-catch。

### H-7：前端 API 响应类型断言绕过类型检查
- **文件**：所有 `frontend/src/api/*.ts`
- **问题**：所有 API 调用使用 `as Promise<T>` 断言，完全绕过 TypeScript 类型检查。后端返回格式变更时不会报错，运行时会崩溃。
- **触发场景**：后端接口返回格式变更时。
- **修复**：使用运行时验证（如 zod）或在响应拦截器中校验。

### H-8：`delete_audio_file` 路径遍历防护在 Windows 上不可靠
- **文件**：`backend/app/services/audio_service.py`
- **问题**：`Path.resolve()` 对不存在的路径在 Windows 上行为不一致，`is_relative_to` 需要 Python 3.9+。
- **触发场景**：攻击者构造恶意 file_path 参数。
- **修复**：始终取 basename 后拼接，使用 `os.path.commonpath` 替代。

## 三、Medium 级别（可能导致功能降级/体验问题）

### M-1：`text_splitter` 中 `segment_size` 可能为 0 导致无限循环
- **文件**：`backend/app/services/text_splitter.py`
- **问题**：`segment_size <= 0` 时，`remaining[:0]` 为空，`cut_pos = 0`，导致无限循环。
- **触发场景**：`AI_SEGMENT_SIZE` 设置为 0 或负数。
- **修复**：添加 `segment_size = max(segment_size, 100)`。

### M-2：`get_setting` 每次调用都查数据库
- **文件**：`backend/app/core/config.py`
- **问题**：每次 AI 调用查询 5 次数据库获取配置，高频调用时产生大量重复查询。
- **触发场景**：批量 AI 操作时。
- **修复**：使用请求级缓存或一次性读取所有配置。

### M-3：`true_false` 题型答案校验不灵活
- **文件**：`backend/app/api/routes/practice.py`
- **问题**：判断题答案使用纯文本匹配，用户输入"正确"但答案是"A"时判定为错误。
- **触发场景**：用户回答判断题时使用了与标准答案语义相同但字面不同的回答。
- **修复**：对 `true_false` 做特殊映射（`A/正确/True` 等价）。

### M-4：`tag` 过滤使用 `ilike` 可能误匹配
- **文件**：`backend/app/api/routes/knowledge_points.py`
- **问题**：JSON 数组字段使用 `ilike` 匹配，搜索"法"会匹配"法律"和"民法"，但也可能匹配到引号等字符。
- **触发场景**：标签包含 SQL 通配符 `%` 或 `_` 时。
- **修复**：解析 JSON 后在 Python 层面过滤，或转义通配符。

### M-5：`ElMessage` 重复调用（前端）
- **文件**：`frontend/src/views/SettingsView.vue`
- **问题**：AI 测试结果既通过 `ElMessage` 弹窗显示，又在页面内显示，同一条消息显示两次。
- **修复**：移除其中一个显示方式。

### M-6：`formatDuration` 重复定义（前端）
- **文件**：`frontend/src/views/PracticeView.vue` 和 `PracticeSessionsView.vue`
- **问题**：两个文件中定义了完全相同的 `formatDuration` 函数。
- **修复**：提取到 `utils/` 中复用。

### M-7：`formatFileSize` 行为不一致（前端）
- **文件**：`frontend/src/views/AudioView.vue` 和 `SettingsView.vue`
- **问题**：AudioView 对 0 字节返回 "-"，SettingsView 返回 "0 B"。
- **修复**：统一实现。

### M-8：路由参数 `id` 未验证（前端）
- **文件**：`frontend/src/views/KnowledgePointDetailView.vue`
- **问题**：`route.params.id` 直接 `Number()` 转换，非数字参数会得到 `NaN`，向后端发送无效请求。
- **触发场景**：用户手动在地址栏输入非数字 ID。
- **修复**：添加 `Number.isFinite(id) && id > 0` 验证。

### M-9：`resetPractice` 不完整（前端）
- **文件**：`frontend/src/views/PracticeView.vue`
- **问题**：`resetPractice` 重置本地状态时，如果 `finishCurrentSession` 的异步操作正在进行，可能覆盖已重置的状态。
- **触发场景**：最后一题提交后网络延迟较高时快速点击"重新开始"。
- **修复**：添加 `isResetting` 标志，在 `finishCurrentSession` 中检查。

### M-10：`WrongQuestionView.vue` 直接修改 props 数据（前端）
- **文件**：`frontend/src/views/WrongQuestionView.vue`
- **问题**：`handleMarkMastered` 直接修改 `wq.is_mastered = true`，不是最佳实践。
- **修复**：通过替换数组中的对象来更新状态。

### M-11：`answerResult` 类型为 `any`（前端）
- **文件**：`frontend/src/views/PracticeView.vue`
- **问题**：`answerResult` 声明为 `ref<any>(null)`，完全丢失类型安全。
- **修复**：使用 `AnswerResult | null` 类型。

### M-12：`filterKB` 变化不触发重新获取（前端）
- **文件**：`frontend/src/views/KnowledgePointListView.vue`
- **问题**：切换知识库筛选器只过滤已加载数据，不会从后端重新获取。
- **修复**：在 `filterKB` 变化时调用 `fetchData()` 并传入 `knowledge_base_id`。

### M-13：`parseApiDate` 时区处理问题（前端）
- **文件**：`frontend/src/utils/date.ts`
- **问题**：对没有时区的日期字符串追加 `Z` 强制转为 UTC，如果后端返回本地时间会导致时间偏移。
- **修复**：根据后端实际返回格式调整解析逻辑。

### M-14：`Question.answer` 字段长度限制
- **文件**：`backend/app/models/question.py`
- **问题**：`answer` 字段为 `String(100)`，对于多选题或长答案可能不够。
- **修复**：改为 `Text` 或增加到 `String(500)`。

### M-15：`generate_collection_filename` 文件名过长
- **文件**：`backend/app/services/audio_service.py`
- **问题**：批量生成音频时，文件名可能超过 255 字符限制。
- **修复**：使用 ID 的哈希值代替完整 ID 列表。

### M-16：`ReviewTask.source` CheckConstraint 可能不完整
- **文件**：`backend/app/models/review_task.py`
- **问题**：CheckConstraint 限制了 source 值，但 service 层可能传入不在约束中的值。
- **修复**：确认所有可能的 source 值都包含在约束中。

### M-17：`.env` 手动解析无法处理特殊字符
- **文件**：`backend/app/core/config.py`
- **问题**：手动逐行解析 .env，无法处理带引号的值、行内注释、多行值等。
- **修复**：使用 `python-dotenv` 的 `load_dotenv()`。

### M-18：`DATABASE_URL` 在 `.env.example` 中但未被使用
- **文件**：`.env.example` + `backend/app/core/database.py`
- **问题**：`.env.example` 定义了 `DATABASE_URL`，但 `database.py` 硬编码了路径。
- **修复**：要么读取环境变量，要么从 `.env.example` 中移除。
