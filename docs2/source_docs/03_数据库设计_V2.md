# 03_数据库设计_V2

# AI 知识点学习与音频播报系统 - V2 数据库设计

## 1. 文档目的

本文档用于说明 V2 新增功能需要的数据库结构。

V2 不重建 V1 数据库，不修改 V1 表的核心含义。

V2 在 V1 基础上新增：

```text
review_tasks 复习任务表
flashcards 闪卡表
```

V2 初期不建议新增：

```text
study_stats 学习统计表
export_tasks 导出任务表
```

原因：

```text
学习统计可以从现有表实时聚合。
导出功能初期可以直接 StreamingResponse 下载，不需要任务表。
```

# 5. 表结构详情

## 5.1 review_tasks 复习任务表

### 5.1.1 作用

保存每日复习任务。

用于：

```text
今日待复习
逾期复习
复习完成记录
下一次复习时间
学习统计
```

### 5.1.2 字段设计

```text
id                integer primary key
knowledge_base_id integer nullable
knowledge_point_id integer nullable
wrong_question_id integer nullable
task_type         string not null
source_type       string not null
status            string not null default pending
priority          integer not null default 0
difficulty_level  integer not null default 0
review_count      integer not null default 0
success_count     integer not null default 0
fail_count        integer not null default 0
due_at            datetime not null
completed_at      datetime nullable
next_review_at    datetime nullable
last_result       string nullable
note              text nullable
created_at        datetime not null
updated_at        datetime not null
```

### 5.1.3 字段说明

```text
knowledge_base_id 方便按知识库筛选。
knowledge_point_id 关联知识点复习任务。
wrong_question_id 关联错题复习任务。
task_type 表示任务类型。
source_type 表示任务来源。
status 表示任务状态。
priority 用于排序。
difficulty_level 用于简单间隔算法。
due_at 表示应复习时间。
completed_at 表示完成时间。
next_review_at 表示下次复习时间。
last_result 保存 again / hard / good / easy。
```

### 5.1.4 task_type 可选值

```text
knowledge_point 知识点复习
wrong_question 错题复习
flashcard 闪卡复习
audio 音频复习
```

V2 初期只需要：

```text
knowledge_point
wrong_question
```

### 5.1.5 source_type 可选值

```text
wrong_question 错题生成
high_importance 高重要程度知识点
new_knowledge 新知识点
manual 手动添加
flashcard 闪卡生成
```

### 5.1.6 status 可选值

```text
pending 待复习
completed 已完成
snoozed 稍后复习
skipped 已跳过
cancelled 已取消
```

### 5.1.7 last_result 可选值

```text
again 完全不会
hard 较难
good 掌握
easy 熟练
```

### 5.1.8 约束建议

```text
knowledge_point_id 和 wrong_question_id 至少一个不为空
status 默认 pending
review_count 默认 0
success_count 默认 0
fail_count 默认 0
priority 默认 0
```

V2 初期可以不做复杂数据库 CheckConstraint，放到 service 层校验。

### 5.1.9 索引建议

```text
idx_review_tasks_status_due_at
idx_review_tasks_knowledge_base_id
idx_review_tasks_knowledge_point_id
idx_review_tasks_wrong_question_id
idx_review_tasks_next_review_at
```

### 5.1.10 SQLAlchemy 模型建议

```python
class ReviewTask(Base):
    __tablename__ = "review_tasks"

    id = Column(Integer, primary_key=True, index=True)
    knowledge_base_id = Column(Integer, ForeignKey("knowledge_bases.id"), nullable=True)
    knowledge_point_id = Column(Integer, ForeignKey("knowledge_points.id"), nullable=True)
    wrong_question_id = Column(Integer, ForeignKey("wrong_questions.id"), nullable=True)

    task_type = Column(String(50), nullable=False)
    source_type = Column(String(50), nullable=False)
    status = Column(String(20), nullable=False, default="pending")

    priority = Column(Integer, nullable=False, default=0)
    difficulty_level = Column(Integer, nullable=False, default=0)
    review_count = Column(Integer, nullable=False, default=0)
    success_count = Column(Integer, nullable=False, default=0)
    fail_count = Column(Integer, nullable=False, default=0)

    due_at = Column(DateTime, nullable=False)
    completed_at = Column(DateTime, nullable=True)
    next_review_at = Column(DateTime, nullable=True)
    last_result = Column(String(20), nullable=True)
    note = Column(Text, nullable=True)

    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)
```

## 5.3 audio_files 表 V2 扩展

### 5.3.1 扩展原因

V1 只做单个知识点音频。

V2 需要支持：

```text
多知识点合集音频
每日复习音频
错题复习音频
```

### 5.3.2 推荐新增字段

```text
audio_type string nullable
source_ids text nullable
title string nullable
```

### 5.3.3 audio_type 可选值

```text
knowledge_point 单个知识点音频
collection 多知识点合集音频
daily_review 每日复习音频
wrong_question 错题复习音频
```

### 5.3.4 source_ids 格式

使用 JSON 字符串：

```json
[1, 2, 3]
```

说明：

```text
audio_type = collection 时，source_ids 保存 knowledge_point_ids。
audio_type = daily_review 时，source_ids 保存 review_task_ids。
audio_type = wrong_question 时，source_ids 保存 wrong_question_ids。
```

### 5.3.5 兼容策略

已有 V1 数据：

```text
audio_type 为空时，前端按 knowledge_point 处理。
```

新增数据：

```text
必须写入 audio_type。
```

### 5.3.6 是否必须改表

推荐改。

如果暂时不改，也可以：

```text
knowledge_point_id 为空表示非单点音频。
text_content 保存音频来源说明。
```

但后续统计和筛选会不方便。

# 7. 复习任务生成规则

## 7.1 候选来源

```text
1. wrong_questions 中 mastered = false 的错题
2. knowledge_points 中 importance = high 的知识点
3. 最近新增且没有 answer_records 的知识点
4. 已有 review_tasks 中 next_review_at <= now 的任务
```

## 7.2 去重规则

同一天内：

```text
同一 knowledge_point_id 不重复生成 pending 任务
同一 wrong_question_id 不重复生成 pending 任务
```

## 7.3 优先级规则

```text
错题复习 priority = 100
high 重要知识点 priority = 80
逾期任务 priority = 70
新知识点 priority = 50
手动添加 priority = 90
```

## 7.4 数量限制

默认：

```text
每日最多 30 个任务
```

从配置读取：

```text
REVIEW_DAILY_LIMIT
```

# 8. 简单间隔算法

V2 初期使用简单规则。

用户完成复习时选择：

```text
again
hard
good
easy
```

对应下次复习：

```text
again: +1 day
hard:  +2 days
good:  +4 days
easy:  +7 days
```

连续 good / easy 后可以放大间隔：

```text
第 1 次 good: +4 days
第 2 次 good: +7 days
第 3 次 good: +15 days
```

后续再升级：

```text
FSRS
```

# 9. 统计查询规则

V2 初期不建统计表。

### 9.1 首页统计

从以下表聚合：

```text
knowledge_points
questions
answer_records
wrong_questions
audio_files
review_tasks
```

### 9.2 知识库掌握度

简单计算：

```text
掌握度 = 已掌握知识点数 / 知识点总数
```

已掌握知识点可先用：

```text
有关联题目且最近答对次数 > 答错次数
或
相关错题 mastered = true
或
复习任务最近结果为 good / easy
```

V2 初期可以先返回粗略值。

### 9.3 正确率

```text
正确率 = correct answer_records / all answer_records
```

按日期筛选：

```text
created_at >= 今日 00:00:00
```

# 15. 数据库迁移注意事项

V1 文档中暂不引入 Alembic。

V2 如果仍然使用：

```text
Base.metadata.create_all
```

注意：

```text
create_all 只会创建新表，不会自动给旧表添加新字段。
```

因此：

```text
review_tasks / flashcards 新表可以直接 create_all。
audio_files 新增字段需要手动 ALTER TABLE 或暂不改旧表。
```

推荐 V2 初期：

```text
优先新增表。
少改旧表。
audio_files 扩展如果暂时不做迁移，就用兼容策略。
```

# 16. V2 数据库验收标准

通过标准：

```text
1. 启动后自动创建 review_tasks 表
2. 做闪卡阶段时自动创建 flashcards 表
3. V1 旧数据不丢失
4. V1 主流程仍可写入原表
5. 可以生成 pending 复习任务
6. 完成复习后 status 变为 completed
7. next_review_at 正确写入
8. 首页统计能从现有表聚合
9. 批量音频不破坏单知识点音频记录
10. app_settings 能保存 V2 新配置
```

不通过条件：

```text
1. 删除或重建 data/app.db
2. 修改 V1 表导致旧功能不可用
3. 批量生成重复 review_tasks
4. API Key 被写入普通业务表
5. data/ 或 app.db 出现在 git 待提交列表
```

---

# 17. 总结

V2 数据库设计建议：

```text
第一步只新增 review_tasks。
第二步根据需要扩展 audio_files 或用兼容策略。
第三步做闪卡时新增 flashcards。
统计先聚合，不急着建 study_stats。
导出先直接下载，不急着建 export_tasks。
AI 日志默认不建，除非需要排错。
```

这样可以最大程度保持 V1 稳定，同时为复习计划、学习统计、批量音频、闪卡提供足够的数据基础。
