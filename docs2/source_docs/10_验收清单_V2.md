# AI 知识点学习与音频播报系统 - V2 验收清单

## 1. 文档目的

本文档用于 V2 阶段验收。

V2 验收必须先确认：

```text
V1 主流程不被破坏
```

再检查：

```text
首页统计
复习计划
批量音频
每日复习音频
错题音频
AI 分段和去重
闪卡
导出
设置
```

---

## 2. 验收总原则

```text
每个阶段结束后检查
最终验收时完整检查
发现问题只做最小修复
不在验收阶段新增大功能
```

不通过即停止继续开发。

# 4. V1 回归验收

V2 验收前必须跑 V1 主流程。

## 4.1 V1 主流程

步骤：

```text
1. 启动后端
2. 启动前端
3. 打开首页
4. 新建知识库
5. 导入资料
6. 保存并提取知识点
7. 查看知识点列表
8. 打开知识点详情
9. 生成题目
10. 打开刷题页面
11. 完成答题
12. 故意答错一道题
13. 打开错题本
14. 标记已掌握
15. 生成单个知识点音频
16. 打开音频播报页面
17. 播放音频
18. 打开设置页
19. 保存设置
20. 重启前后端
21. 确认数据仍存在
```

通过标准：

```text
所有 V1 页面可用。
V1 数据能保存。
V1 AI 和 TTS 可用。
V1 音频可播放。
```

不通过：

```text
先修 V1，不继续 V2。
```

# 14. AI 长文本分段验收

## 14.1 短文本不变

步骤：

```text
1. 导入短资料
2. 点击保存并提取知识点
```

通过标准：

```text
仍按 V1 流程提取。
split_used=false 或不显示分段。
```

## 14.2 长文本分段

步骤：

```text
1. 导入超过 AI_SEGMENT_SIZE 的资料
2. 点击保存并提取知识点
```

通过标准：

```text
自动分段。
返回 segment_count。
新增知识点。
重复知识点被跳过。
前端显示 created_count / skipped_count。
```

## 14.3 超过分段上限

步骤：

```text
1. 导入极长资料
2. 触发超过 AI_MAX_SEGMENTS
```

通过标准：

```text
提示资料过长，自动分段超过限制，请拆分后导入。
```

# 15. AI 去重和 JSON 修复验收

## 15.1 知识点去重

步骤：

```text

2. 触发 AI 提取或模拟返回
```

通过标准：

```text

```

## 15.2 题目去重

步骤：

```text

```

通过标准：

```text
重复 stem 不重复入库。

```

## 15.3 JSON 修复

步骤：

```text
1. 模拟 AI 返回 Markdown JSON 或轻微格式错误
2. 触发解析
```

通过标准：

```text
系统最多修复 1 次。
修复成功则继续保存。
修复失败则明确提示。
不会无限重试。
```

# 20. 错误处理检查

必须有明确提示：

```text
今日没有可生成的复习任务
复习任务不存在
请选择至少 1 个知识点
单次最多选择 N 个知识点
批量音频文本过长，请减少知识点数量
资料过长，自动分段超过限制，请拆分后导入
AI 返回格式异常，已尝试修复但仍失败
导出数据为空
设置值超出允许范围
```

前端要求：

```text
不显示空白错误
不显示后端堆栈
不重复弹多个错误
```
