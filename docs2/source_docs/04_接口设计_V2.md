# 04_接口设计_V2

# AI 知识点学习与音频播报系统 - V2 接口设计

## 1. 文档目的

本文档定义 V2 新增功能的 API 接口。

V2 接口必须兼容 V1：

```text
不删除 V1 接口
不改变 V1 返回结构的核心字段
新增接口统一放在 /api 下
前端统一通过 frontend/src/api/*.ts 调用
```

---

## 2. 接口基础约定

### 2.1 后端基础地址

```text
http://localhost:8000
```

### 2.2 API 前缀

```text
/api
```

### 2.3 请求格式

默认：

```text
Content-Type: application/json
```

导出接口例外：

```text
text/csv
text/markdown
application/octet-stream
```

### 2.4 返回格式

普通接口返回 JSON。

成功示例：

```json
{
  "message": "操作成功",
  "data": {}
}
```

列表接口可以直接返回数组，也可以返回分页结构。

V2 初期推荐直接返回数组，保持简单。

### 2.5 错误格式

FastAPI 默认：

```json
{
  "detail": "错误信息"
}
```

### 2.6 时间格式

继续使用 ISO 字符串：

```text
2026-05-07T10:30:00
```

### 2.7 分页约定

V2 初期列表可以不分页。

如果加分页，统一使用：

```text
page
page_size
```

返回：

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "page_size": 20
}
```

## 4.1 获取复习任务列表

```http
GET /api/review/tasks
```

```text
knowledge_base_id 可选
status 可选 pending / completed / snoozed / skipped / cancelled
task_type 可选 knowledge_point / wrong_question / flashcard / audio
```

```json
[
  {
    "id": 1,
    "knowledge_base_id": 1,
    "knowledge_point_id": 12,
    "wrong_question_id": null,
    "task_type": "knowledge_point",
    "source_type": "high_importance",
    "status": "pending",
    "priority": 80,
    "due_at": "2026-05-07T08:00:00",
    "completed_at": null,
    "next_review_at": null,
    "last_result": null,
    "review_count": 0,
    "success_count": 0,
    "fail_count": 0,
    "knowledge_point_title": "政府社会职能",
    "knowledge_base_name": "三支一扶",
    "created_at": "2026-05-07T08:00:00",
    "updated_at": "2026-05-07T08:00:00"
  }
]
```

### 前端用途

```text
ReviewView 任务列表
DashboardView 今日待复习数量
```

---

## 4.2 生成今日复习任务

### 请求

```http
POST /api/review/tasks/generate
```

### 请求体

```json
{
  "knowledge_base_id": 1,
  "limit": 30,
  "include_wrong_questions": true,
  "include_high_importance": true,
  "include_new_knowledge": true
}
```

字段说明：

```text
knowledge_base_id 可选，为空表示所有知识库
limit 可选，默认读取 REVIEW_DAILY_LIMIT
include_wrong_questions 是否包含错题
include_high_importance 是否包含高重要知识点
include_new_knowledge 是否包含新知识点
```

### 返回示例

```json
{
  "created_count": 18,
  "skipped_count": 5,
  "message": "今日复习任务已生成"
}
```

### 常见错误

```json
{
  "detail": "今日没有可生成的复习任务"
}
```

### 规则

```text
同一知识点当天不重复生成 pending 任务。
同一错题当天不重复生成 pending 任务。
limit 不能超过配置上限。
```

## 4.4 完成复习任务

### 请求

```http
POST /api/review/tasks/{task_id}/complete
```

### 请求体

```json
{
  "result": "good",
  "note": "已掌握"
}
```

result 可选：

```text
again
hard
good
easy
```

### 返回示例

```json
{
  "id": 1,
  "status": "completed",
  "last_result": "good",
  "review_count": 1,
  "success_count": 1,
  "fail_count": 0,
  "completed_at": "2026-05-07T10:30:00",
  "next_review_at": "2026-05-11T10:30:00",
  "message": "复习任务已完成"
}
```

### 规则

```text
again / hard 计入 fail_count。
good / easy 计入 success_count。
根据 result 计算 next_review_at。
```

## 7.1 批量生成知识点音频

# 8. AI 增强接口

V2 优先扩展已有接口，不新增过多 AI 路由。

## 8.1 长文本分段提取

### 请求

```http
POST /api/materials/import-and-extract
```

### V2 新增请求字段

```json
{
  "knowledge_base_id": 1,
  "title": "长篇资料",
  "content": "很长的资料正文...",
  "source": "教材",
  "note": "",
  "enable_split": true
}
```

### V2 返回新增字段

```json
{
  "material_id": 10,
  "knowledge_points": [],
  "split_used": true,
  "segment_count": 3,
  "created_count": 12,
  "skipped_count": 2
}
```

### 规则

```text
enable_split 默认读取 AI_ENABLE_LONG_TEXT_SPLIT。
segment_count 不能超过 AI_MAX_SEGMENTS。
重复知识点跳过。
```

---

## 8.2 JSON 修复

不建议暴露独立前端接口。

后端内部流程：

```text
AI 返回解析失败
↓
如果 AI_ENABLE_JSON_REPAIR=true
↓
自动调用修复 Prompt 一次
↓
成功则继续保存
↓
失败则返回错误
```

错误示例：

```json
{
  "detail": "AI 返回格式异常，已尝试修复但仍失败"
}
```

通过标准：

```text
1. V1 接口仍可用
2. /api/review/tasks 能返回复习任务
3. /api/review/tasks/generate 能生成今日任务
4. /api/review/tasks/{id}/complete 能完成任务并写入 next_review_at
5. /api/stats/overview 能返回首页增强统计
6. /api/tts/generate-batch 有数量限制
7. /api/audio-files 能兼容旧音频和新音频
8. /api/materials/import-and-extract 支持 enable_split
9. 导出接口能下载 CSV
10. 设置接口能保存 V2 新配置
```

不通过条件：

```text
1. 改坏 V1 页面调用的字段
2. 批量接口没有上限
3. AI / TTS 错误返回空白或 500 堆栈
4. API Key 明文返回给前端
5. 导出接口把临时文件写入 data/ 且不清理
```

---

# 16. 总结

V2 接口优先级：

```text
1. /api/stats/overview
2. /api/review/tasks
3. /api/review/tasks/generate
4. /api/review/tasks/{id}/complete
5. /api/tts/generate-batch
6. /api/materials/import-and-extract enable_split
7. /api/export/*.csv
8. /api/flashcards
```

建议先做统计和复习，再做批量音频，最后做闪卡和导出。
