# AI 知识点学习与音频播报系统 - V2 开发阶段计划

## 1. 文档目的

本文档用于把 V2 拆成可执行开发阶段。

V2 必须建立在 V1 已完成基础上。

V2 开发原则：

```text
先验收 V1
再加 V2
每次只做一个阶段
每个阶段都要能运行
不一次性重构
不破坏 V1 主流程
```

---

## 2. 总体开发原则

### 2.1 先收口，再扩展

开始 V2 前必须先做：

```text
V1 最终验收
V1 bug 修复
V1 一键启动确认
```

不能跳过。

### 2.2 每次只做一个阶段

每个阶段只完成本阶段目标。

不要提前开发后续功能。

### 2.3 每个阶段都要验收

阶段完成后检查：

```text
后端能启动
前端能启动
相关页面能打开
相关接口能调用
V1 主流程不被破坏
```

### 2.4 控制复杂度

V2 不引入：

```text
登录注册
多用户
云同步
Docker
复杂 RAG
后台任务队列
大型状态管理库
```

---

## 3. 阶段总览

V2 阶段从 V1 后继续编号：

```text
阶段 19：V1 最终验收和问题修复
阶段 20：首页统计增强
阶段 21：复习计划数据表和后端接口
阶段 22：复习计划前端页面
阶段 23：学习统计接口和页面
阶段 24：批量知识点音频
阶段 25：每日复习音频和错题音频
阶段 26：AI 长文本分段
阶段 27：AI 去重和 JSON 修复
阶段 28：知识点闪卡
阶段 29：导入导出
阶段 30：V2 整体联调和最终验收
```

推荐优先做：

```text
阶段 19 → 20 → 21 → 22 → 24 → 30
```

闪卡、导出可以后置。

# 17. 总结

V2 不应一次性全做。

最小可用 V2：

```text

复习计划
批量知识点音频
```

```text
生成
练习

统计
收听
```

再逐步补：

```text
AI 稳定性
闪卡
导出
```
