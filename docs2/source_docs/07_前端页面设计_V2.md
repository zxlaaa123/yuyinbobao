# AI 知识点学习与音频播报系统 - V2 前端页面设计

## 1. 文档目的

本文档定义 V2 前端页面和交互设计。

V2 不重做 V1 前端，不改变现有页面核心职责。

V2 在现有页面基础上增强：

```text
DashboardView
KnowledgePointListView
KnowledgePointDetailView
WrongQuestionView
AudioView
SettingsView
```

V2 新增页面：

```text
ReviewView
StatsView
FlashcardView
```

V2 初期建议只新增：

```text
ReviewView
```

统计先集成到 DashboardView。

---

## 2. 前端设计目标

V2 前端目标：

```text
1. 补齐复习闭环
2. 展示学习进度
3. 支持批量音频
4. 降低 AI / TTS 长任务误操作
5. 保持 V1 页面简单可用
```

V2 不追求：

```text
复杂动画
营销式首页
过度图表化
移动端深度适配
```

---

## 3. UI 风格

继续沿用 V1：

```text
Element Plus
左侧菜单
顶部内容区
表格 + 表单 + 弹窗

```

V2 重点：

```text

状态更清楚

```

## 7. 通用交互规则

### 7.1 Loading

以下操作必须 loading：

```text
生成今日复习任务
完成复习任务
批量生成音频
生成每日复习音频
生成错题复习音频
长文本分段提取
AI 生成闪卡
导出 CSV
保存设置
```

### 7.2 禁用重复点击

loading 时按钮禁用。

### 7.3 二次确认

以下操作需要确认：

```text
删除复习任务
删除闪卡
删除音频
批量生成音频
长文本分段提取
```

批量生成音频确认文案：

```text
将为 12 个知识点生成合集音频，可能需要较长时间，是否继续？
```

### 7.4 空状态

所有列表必须有空状态：

```text
暂无复习任务
暂无统计数据
暂无闪卡
暂无该类型音频
```

## 8.8 SettingsView 增强

### 目标

管理 V2 新配置。

### 新增分区

```text
复习设置
AI 增强设置
TTS 批量设置
导出设置
```

### 复习设置

字段：

```text
每日复习上限 REVIEW_DAILY_LIMIT
包含已掌握错题 REVIEW_INCLUDE_MASTERED_WRONG
优先高重要知识点 REVIEW_PRIORITY_HIGH_IMPORTANCE
```

### AI 增强设置

字段：

```text
启用长文本分段 AI_ENABLE_LONG_TEXT_SPLIT
最大分段数 AI_MAX_SEGMENTS
单段长度 AI_SEGMENT_SIZE
启用 JSON 修复 AI_ENABLE_JSON_REPAIR
保存 AI 原始返回 AI_SAVE_RAW_RESPONSE
```

### TTS 批量设置

字段：

```text
批量最大知识点数 TTS_BATCH_MAX_POINTS
批量最大文本长度 TTS_BATCH_MAX_CHARS
语速 TTS_SPEED
格式 TTS_FORMAT
音色 XIAOMI_TTS_VOICE
```

### 导出设置

字段：

```text
最大导出行数 EXPORT_MAX_ROWS
导出是否包含解析 EXPORT_INCLUDE_EXPLANATION
```

### 接口

```text
GET /api/settings
PUT /api/settings
```

### 校验

```text
数值字段必须是数字
最大值不能超过后端允许范围
布尔字段用开关
API Key 继续脱敏
```

# 10. 表单校验

### 10.1 复习任务

```text
limit 必须大于 0
limit 不能超过 REVIEW_DAILY_LIMIT
result 必须是 again / hard / good / easy
```

### 10.2 批量音频

```text
必须选择至少 1 个知识点
不能超过 TTS_BATCH_MAX_POINTS
标题可为空，为空时后端自动生成
```

### 10.3 闪卡

```text
front 不为空
back 不为空
card_type 必选
cloze 类型 cloze_text 不为空
```

### 10.4 设置

```text
数值字段必须为数字
布尔字段使用开关
范围错误时阻止提交
```

# 11. 错误提示

前端统一使用：

```text
ElMessage.error(error.response?.data?.detail || "操作失败")
```

常见提示：

```text
今日没有可生成的复习任务
复习任务不存在
请选择至少 1 个知识点
单次最多选择 20 个知识点
批量音频文本过长，请减少知识点数量
AI 返回格式异常，已尝试修复但仍失败
导出数据为空
```

# 14. 前端开发顺序

推荐：

```text
1. stats.ts + DashboardView 增强
2. review.ts + ReviewView
3. AudioView 类型筛选
4. KnowledgePointListView 批量音频
5. SettingsView V2 配置
6. WrongQuestionView 错题音频
7. KnowledgePointDetailView 闪卡入口
8. FlashcardView
9. StatsView
10. export.ts + 导出按钮
```

优先级：

```text
复习计划 > 首页统计 > 批量音频 > 设置增强 > 错题音频 > 闪卡 > 导出 > 独立统计页
```

# 15. 前端验收标准

通过标准：

```text
1. V1 所有页面仍可打开
2. 菜单新增复习计划且路由可访问
3. 首页能显示 V2 统计
4. ReviewView 能生成、查看、完成复习任务
5. 批量生成音频按钮有选择限制和 loading
6. AudioView 能按音频类型筛选
7. SettingsView 能保存 V2 配置
8. 空状态、错误提示完整
9. 删除类操作有确认
10. 浏览器控制台无明显报错
```

不通过条件：

```text
1. 改坏 V1 路由
2. 页面按钮重复点击导致多次 AI / TTS 请求
3. API Key 明文显示
4. 批量操作没有数量提示
5. 表格或弹窗内容严重重叠
```
