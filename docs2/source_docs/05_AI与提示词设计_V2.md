# 05_AI与提示词设计_V2

# AI 知识点学习与音频播报系统 - V2 AI 与提示词设计

## 1. 文档目的

本文档定义 V2 的 AI 能力增强方案。

V2 不替换 V1 AI 功能，而是在已有能力上增强：

```text
AI 知识点提取
AI 题目生成
AI 连接测试
JSON 解析
```

V2 新增：

```text
长文本自动分段
知识点去重
题目去重
JSON 修复
闪卡生成
复习音频播报稿生成
AI 调用成本控制
```

---

## 2. AI 功能定位

V2 AI 继续服务于学习流程：

```text
资料 → 知识点
知识点 → 题目
知识点 → 闪卡
知识点 / 错题 / 复习任务 → 播报稿
```

V2 不做：

```text
复杂 RAG 问答
向量检索
聊天机器人
自动学习规划大模型 Agent
```

---

## 3. AI 架构原则

继续沿用 V1：

```text
OpenAI-compatible Chat Completions
后端统一封装
前端不直接调用 AI
API Key 从 .env 或 app_settings 读取
AI 返回必须后端校验
失败时给前端明确错误
```

V2 新增原则：

```text
1. 长文本先分段，再合并。
2. AI 输出必须结构化 JSON。
3. 解析失败最多修复 1 次。
4. 去重优先用本地规则，不优先调用 AI。
5. 所有批量 AI 操作必须有上限。
6. 默认不保存 AI 原始返回，除非设置开启。
```

---

## 4. V2 AI 功能清单

```text
1. 长文本分段提取知识点
2. AI JSON 修复
3. 知识点去重
4. 题目去重
5. 闪卡生成
6. 复习音频播报稿生成
7. AI 调用日志可选保存
8. AI 成本和数量限制
```

# 6. 长文本分段提取知识点

## 6.1 功能说明

V1 中资料过长时提示用户分段导入。

V2 支持自动分段：

```text
当资料正文超过 AI_SEGMENT_SIZE 时
后端自动切分成多个段落
逐段调用 AI 提取知识点
最后合并、去重、保存
```

## 6.2 输入参数

来自：

```http
POST /api/materials/import-and-extract
```

V2 新增字段：

```json
{
  "enable_split": true
}
```

如果前端不传：

```text
读取 AI_ENABLE_LONG_TEXT_SPLIT 配置。
```

## 6.3 配置项

```text
AI_ENABLE_LONG_TEXT_SPLIT=true
AI_MAX_SEGMENTS=10
AI_SEGMENT_SIZE=7000
```

## 6.4 分段规则

推荐规则：

```text
1. 先按空行切分段落。
2. 段落过长时按句号、问号、感叹号切分。
3. 仍过长时按固定长度切分。
4. 每段尽量不超过 AI_SEGMENT_SIZE。
5. 总段数不能超过 AI_MAX_SEGMENTS。
```

## 6.5 text_splitter.py 设计

建议函数：

```python
def split_text_for_ai(content: str, max_chars: int, max_segments: int) -> list[str]:
    ...
```

返回：

```text
字符串数组
```

异常：

```text
内容为空
分段数超过限制
```

错误提示：

```json
{
  "detail": "资料过长，自动分段超过限制，请拆分后导入"
}
```

## 6.6 分段提取流程

```text
保存 material
↓
检测 content 长度
↓
切分 segments
↓
逐段调用 extract_knowledge_points
↓
合并结果
↓
dedup_service 去重
↓
保存 knowledge_points
↓
返回 split_used / segment_count / created_count / skipped_count
```

## 6.7 返回示例

```json
{
  "material_id": 10,
  "split_used": true,
  "segment_count": 3,
  "created_count": 12,
  "skipped_count": 2,
  "knowledge_points": []
}
```

# 7. 知识点去重

## 7.1 V1 规则

```text
同一 material_id 下 title 完全相同则跳过。
```

## 7.2 V2 规则

V2 增强为：

```text
1. title 标准化后完全相同则跳过。
2. title 相同但 summary 不同时，保留更完整的一条。
3. 同一 material_id 下优先去重。
4. 同一 knowledge_base_id 下可以提示相似，不强制跳过。
```

## 7.3 标准化规则

```text
去除首尾空格
去除所有空白字符
去除常见标点
统一大小写
中文数字暂不转换
```

## 7.4 dedup_service.py 建议函数

```python
def normalize_text(text: str) -> str:
    ...

def dedup_knowledge_points(points: list[dict], existing_titles: list[str]) -> tuple[list[dict], int]:
    ...
```

返回：

```text
去重后的知识点列表
跳过数量
```

## 7.5 不建议 V2 初期做

```text
向量相似度
embedding 去重
AI 判断相似度
跨知识库强制去重
```

# 8. 题目去重

## 8.1 V1 规则

```text

```

## 8.2 V2 规则

```text
4. 生成数量不足时返回 created_count / skipped_count。
```

# 9. AI JSON 修复

## 9.1 功能说明

AI 返回不稳定时，V2 允许自动修复一次 JSON。

适用：

```text
知识点提取
题目生成
闪卡生成
播报稿结构化生成
```

## 9.2 配置项

```text
AI_ENABLE_JSON_REPAIR=true
```

## 9.3 流程

```text
AI 原始返回
↓
json_parser 尝试解析
↓
失败
↓
如果开启 JSON 修复
↓
调用 repair_json
↓
再次解析
↓
成功返回结构化数据
↓
失败抛出错误
```

## 9.4 JSON 修复 System Prompt

```text
你是 JSON 修复工具。
你的任务是把用户提供的内容修复为合法 JSON。
只能输出 JSON。
不要解释。
不要输出 Markdown。
不要添加原文中不存在的业务字段。
如果字段值缺失，使用空字符串、空数组或 null。
```

## 9.5 JSON 修复 User Prompt 模板

```text
请将下面内容修复为合法 JSON。

要求：
1. 只能输出 JSON。
2. 不要输出 Markdown 代码块。
3. 保留原有字段含义。
4. 不要扩写业务内容。

原始内容：
{raw_text}
```

## 9.6 错误提示

```json
{
  "detail": "AI 返回格式异常，已尝试修复但仍失败"
}
```

# 10. 闪卡生成设计

## 10.1 功能说明

根据知识点生成记忆闪卡。

入口：

```http
POST /api/flashcards/generate-from-point/{knowledge_point_id}
```

## 10.2 输入

```text
knowledge_point.title
knowledge_point.summary
knowledge_point.detail
knowledge_point.key_points
knowledge_point.confusing_points
knowledge_point.mnemonics
knowledge_point.examples
```

## 10.3 输出字段

```json
[
  {
    "card_type": "basic",
    "front": "问题",
    "back": "答案",
    "cloze_text": null
  }
]
```

## 10.4 card_type 可选值

```text
basic
reverse
cloze
```

## 10.5 闪卡生成 System Prompt

```text
你是一个严谨的学习卡片生成助手。
你的任务是根据知识点生成适合复习的闪卡。

要求：
1. 只能基于给定知识点生成，不要编造外部内容。
2. 每张卡片只考一个明确知识。
3. 正面问题必须清晰。
4. 背面答案必须简洁准确。
5. cloze 类型必须提供挖空文本。
6. 不要生成过于宽泛的问题。
7. 只能输出 JSON 数组。
8. 不要输出 Markdown。
```

## 10.6 闪卡生成 User Prompt 模板

```text
请根据下面知识点生成 {count} 张闪卡。

允许的卡片类型：
{card_types}

知识点标题：
{title}

摘要：
{summary}

详细解释：
{detail}

考点：
{key_points}

易混点：
{confusing_points}

记忆方法：
{mnemonics}

示例：
{examples}

返回 JSON 数组，格式如下：
[
  {
    "card_type": "basic",
    "front": "问题",
    "back": "答案",
    "cloze_text": null
  }
]
```

## 10.7 闪卡质量要求

```text
front 不少于 4 个字
back 不为空
basic 卡必须是问答结构
reverse 卡不能和 basic 完全重复
cloze 卡必须包含挖空标记
每张卡只考一个点
```

# 11. 复习音频播报稿生成

## 11.1 功能说明

V1 TTS 文本主要来自单个知识点。

V2 需要生成：

```text
多知识点合集播报稿
每日复习播报稿
错题复习播报稿
```

## 11.2 是否必须调用 AI

不一定。

推荐策略：

```text
优先使用后端模板拼接。
只有需要更自然的讲解稿时再调用 AI。
```

原因：

```text
省成本
稳定
不容易跑题
```

## 11.3 模板拼接规则

多知识点合集：

```text
本期学习音频包含 {count} 个知识点。

第 1 个知识点：{title}
摘要：{summary}
考点：{key_points}
易混点：{confusing_points}
记忆方法：{mnemonics}

第 2 个知识点：...
```

错题复习：

```text
下面开始错题复习。

第 1 道错题：
题目：{stem}
你的错误答案：{user_answer}
正确答案：{answer}
解析：{explanation}
关联知识点：{knowledge_point_title}
```

## 11.4 AI 优化播报稿 Prompt 可选

System Prompt：

```text
你是学习音频播报稿整理助手。
你的任务是把结构化知识点整理成适合 TTS 朗读的中文播报稿。

要求：
1. 不要添加外部知识。
2. 不要改变原意。
3. 语言自然，适合朗读。
4. 保留知识点编号。
5. 不要输出 Markdown。
```

User Prompt：

```text
请将下面内容整理成适合 TTS 朗读的播报稿。

内容：
{content}
```

V2 初期建议不启用。

# 19. V2 AI 验收标准

通过标准：

```text
1. V1 AI 知识点提取仍可用
2. V1 AI 题目生成仍可用
3. 长文本能自动分段提取
4. 分段数超过限制时返回明确错误
5. 重复知识点能跳过
6. 重复题目能跳过
7. AI JSON 解析失败时最多修复 1 次
8. 修复失败时返回明确错误
9. 能从知识点生成闪卡
10. AI 批量操作有数量上限
```

不通过条件：

```text
1. 前端直接调用 AI 外部接口
2. API Key 写入前端代码
3. JSON 修复无限重试
4. 长文本无限分段
5. AI 生成内容未经校验直接入库
6. 生成重复内容大量入库
```

# 21. 总结

V2 AI 的核心不是增加更多模型能力，而是让现有 AI 功能更稳定：

```text
长文本能处理
重复内容少入库
JSON 错误能修复
批量调用有上限
知识点能转闪卡
音频播报稿更适合复习
```

建议先实现：

```text
去重
长文本分段
JSON 修复
```

再实现：

```text
闪卡生成
播报稿润色
AI 日志
```
