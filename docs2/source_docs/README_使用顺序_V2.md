# README_使用顺序_V2

# AI 知识点学习与音频播报系统 - V2 文档使用顺序

## 1. 项目定位

本项目 V1 已完成基础学习闭环：

```text
资料导入
AI 提取知识点
知识点管理
AI 生成题目
刷题练习
错题本
TTS 音频生成
音频播放
系统设置
首页统计
```

V2 的目标不是重做系统，而是增强学习效果：

```text
复习计划
学习统计
批量音频
AI 稳定性
闪卡
导出
```

---

## 2. V2 与 V1 的关系

V2 必须建立在 V1 上。

V2 不能破坏：

```text
知识库
资料导入
AI 知识点提取
知识点详情
AI 题目生成
刷题练习
错题本
单个知识点音频
音频播放
设置页
```

V2 继续遵守 V1 不做内容：

```text
登录注册
多用户权限
云同步
Docker
手机 App
浏览器插件
支付系统
复杂 RAG
PDF 深度解析
Word 深度解析
OCR
桌面打包
```

---

## 3. 推荐技术栈

继续使用 V1 技术栈：

```text
前端：Vue 3 + TypeScript + Vite + Element Plus
后端：FastAPI + SQLAlchemy + Pydantic
数据库：SQLite
AI：OpenAI-compatible Chat Completions
TTS：Mock / 小米 TTS / 可扩展 Provider
本地文件：data/
```

V2 不新增复杂基础设施。

## 7. 开发优先级

始终按这个顺序判断：

```text
V1 不回归 > 能运行 > 能保存 > 能复习 > 能统计 > 能批量听 > AI 更稳定 > 能导出 > 好看
```

不要为了复杂而复杂。

不建议优先做：

```text
闪卡完整复习算法
Anki apkg
复杂图表
后台任务队列
向量检索
移动端适配
```

---

## 8. 给 AI 编程工具的总提示词

可以直接复制：

```text
请先阅读 docs/README_使用顺序_精简版.md、docs/10_验收清单.md，以及 docs2/README_使用顺序_V2.md。

当前目标是开发 V2。

重要规则：
1. V2 必须建立在 V1 之上。
2. 任何阶段开始前，不能破坏 V1 主流程。
3. 每次只做 docs2/08_开发阶段计划_V2.md 中的一个阶段。
4. 不要提前开发后续阶段。
5. 不要做登录注册、多用户、云同步、Docker、复杂 RAG。
6. 前端不能直接调用 AI 或 TTS 外部接口。
7. API Key 不能写死到代码。
8. 批量 AI / TTS 操作必须有上限。
9. 本地数据继续保存在 data/。
10. 修改后必须说明改了哪些文件、如何启动、如何测试。

请从阶段 19 开始：V1 最终验收和问题修复。
```

## 9. 每个阶段的推荐要求格式

不要只说：

```text
继续
```

建议这样说：

```text
请开始阶段 20：首页统计增强。
要求：
1. 只做阶段 20。
2. 新增 /api/stats/overview 和 /api/stats/knowledge-bases。
3. 增强 DashboardView。
4. 不新增复习任务表。
5. 不修改 V1 其他页面。
6. 完成后告诉我如何测试首页统计。
```

复习计划阶段示例：

```text
请开始阶段 21：复习计划数据表和后端接口。
要求：
1. 只做后端和数据库。
2. 新增 review_tasks 表。
3. 新增 review 接口。
4. 不做前端页面。
5. 不做复杂 FSRS。
6. 完成后告诉我如何用接口测试。
```

## 10. V2 成功标准

V2 成功标准：

```text
1. V1 主流程仍可完整跑通。
2. 首页能展示学习进度。
3. 用户能生成今日复习任务。
4. 用户能完成复习任务。
5. 用户能选择多个知识点生成合集音频。
6. 音频能正常播放。
7. AI 长文本和重复内容处理更稳定。
8. 设置页能管理 V2 配置。
9. 数据重启后仍存在。
10. .env、data、app.db、音频文件不进入 git。
```

## 11. V2 最小闭环

```text

↓
产生错题

生成复习任务
```

```text
闪卡
导出

```

```text

```

