# 02_技术架构_V2

# AI 知识点学习与音频播报系统 - V2 技术架构设计

## 1. 文档目的

本文档用于说明 V2 功能如何接入现有 V1 架构。

V2 不重构 V1 主体，不更换技术栈，不引入复杂基础设施。

V2 目标：

```text
在现有 FastAPI + Vue3 + SQLite + 本地文件架构上
增加复习计划、学习统计、闪卡、批量音频、AI 稳定性增强、导入导出
```

---

## 2. 架构原则

V2 继续遵守 V1 原则：

```text
本地优先
单用户
SQLite 默认数据库
前端不直接调用 AI / TTS
后端统一封装 AI / TTS
音频文件保存在 data/audio/
配置来自 .env 和 app_settings
接口统一放在 /api 下
```

V2 新增原则：

```text
1. 不破坏 V1 主流程
2. 新功能按模块独立增加
3. 长任务必须有数量限制和 loading
4. AI / TTS 批量操作必须可控
5. 统计优先通过现有表聚合，必要时再加缓存表
6. 复习算法先简单，后续再升级 FSRS
```

---

## 3. 总体架构

V2 架构：

```text
frontend Vue3
  ↓ HTTP

  ↓
services
  ├─ ai_service
  ├─ tts_service
  ├─ review_service
  ├─ stats_service
  ├─ flashcard_service
  ├─ export_service
  └─ setting_service
  ↓
SQLite data/app.db
  ↓
data/audio
data/uploads
data/vector_store
```

说明：

```text
data/vector_store 继续保留，但 V2 不强制使用。
data/uploads 继续保留，但 V2 不做复杂 PDF / Word / OCR。
```

## 7. 后端架构

### 7.1 api 层

api 层只负责：

```text
参数接收
权限以外的基础校验
调用 service
返回统一 JSON
抛出 HTTPException
```

V2 暂不做登录权限。

### 7.2 service 层

service 层负责业务规则：

```text
review_service      复习任务生成和完成
stats_service       统计聚合
flashcard_service   闪卡生成和管理
export_service      CSV / Markdown 导出
text_splitter       长文本切分
dedup_service       知识点和题目去重
```

### 7.3 models 层

新增表建议：

```text
review_tasks
flashcards
```

统计初期不建议新增表。

原因：

```text
answer_records / wrong_questions / knowledge_points 已经能聚合出大部分统计。
```

### 7.4 schemas 层

新增：

```text
review.py
stats.py
flashcard.py
```

导出接口可不单独建 schema，直接返回 StreamingResponse。

## 9. 复习计划架构

### 9.1 数据来源

复习任务从以下数据生成：

```text
wrong_questions
knowledge_points.importance
answer_records
questions
```

### 9.2 生成流程

```text
用户点击生成今日复习
↓
review_service 查询候选知识点和错题
↓
按规则去重
↓
写入 review_tasks
↓
前端展示任务列表
```

### 9.3 完成流程

```text
用户完成复习
↓
提交掌握结果
↓
更新 review_tasks.status
↓
计算 next_review_at
↓
必要时同步 wrong_questions.mastered
```

### 9.4 简单复习算法

V2 初期推荐：

```text
again: next_review_at = now + 1 day
hard:  next_review_at = now + 2 days
good:  next_review_at = now + 4 days
easy:  next_review_at = now + 7 days
```

后续再升级：

```text
FSRS
```

## 12. 批量音频架构

### 12.1 复用 V1

继续复用：

```text
tts_service.py
audio_service.py
audio_files 表
data/audio/
/audio/xxx.mp3 静态访问
```

### 12.2 批量生成流程

```text
前端提交 knowledge_point_ids
↓
后端校验数量上限
↓
查询知识点
↓
生成合集播报文本
↓
调用 TTS
↓
保存音频文件
↓
写入 audio_files
```

### 12.3 音频文件策略

V2 初期推荐：

```text
先生成一个合集文本，再调用一次 TTS。
如果文本超过限制，则拆成多个 audio_files。
前端按顺序播放。
```

暂不强制做：

```text
mp3 合并
后台任务队列
```

### 12.4 数量限制

必须限制：

```text
单次最多 20 个知识点
单次播报文本最多 12000 字
```

具体值可通过 SettingsView 配置。

## 13. AI 架构增强

### 13.1 长文本分段

新增：

```text
text_splitter.py
```

职责：

```text
按长度切分资料正文
尽量按段落切分
避免切断句子
限制最大段数
```

### 13.2 去重服务

新增：

```text
dedup_service.py
```

职责：

```text
normalize_title
normalize_stem
知识点标题去重
题目题干去重
```

V2 初期不用向量相似度。

推荐先做：

```text
去空格
去标点
统一大小写
完全匹配
```

### 13.3 JSON 修复

扩展：

```text
ai_service.py
json_parser.py
prompt_templates.py
```

流程：

```text
解析失败
↓
调用 JSON 修复 Prompt 一次
↓
再次解析
↓
失败则返回错误
```

## 16. API 规则

继续遵守 V1：

```text
成功返回 JSON
失败抛 HTTPException
时间字段使用 ISO 字符串
接口路径使用 /api 前缀
前端统一通过 Axios 封装
```

批量接口必须返回：

```text
成功数量
跳过数量
失败数量
失败原因
```

示例：

```json
{
  "success_count": 12,
  "skipped_count": 2,
  "failed_count": 1,
  "message": "批量音频生成完成"
}
```

## 17. 错误处理

V2 新增常见错误：

```text
复习任务不存在
今日没有可复习内容
批量音频知识点数量超过限制
AI 分段数量超过限制
AI JSON 修复失败
导出数据为空
```

前端提示必须明确：

```text
没有可复习内容
请选择至少 1 个知识点
单次最多选择 20 个知识点
资料过长，请拆分后导入
AI 返回格式异常，已尝试修复但仍失败
```


继续禁止：

```text

```

### 18.3 本地文件

导出文件初期不落盘。

推荐：


## 20. V2 架构验收标准

V2 架构通过标准：

```text
1. V1 所有页面仍能打开
2. V1 主流程仍能跑通
3. 新增 routes 已注册到 api/__init__.py 或 main.py
4. 新增 models 已进入 Base.metadata.create_all
5. 新增前端路由能访问
6. 复习任务能生成和完成
7. 首页统计不破坏原有 dashboard 接口
8. 批量音频有数量限制
9. AI 长文本分段有最大段数限制
10. 所有新增配置可从 app_settings 或 .env 读取
```

不通过条件：

```text
1. 为 V2 引入登录、多用户、云同步
2. 前端直接请求 AI / TTS 外部服务
3. 批量接口无上限
4. AI / TTS Key 明文返回给前端
5. 为统计功能提前引入复杂缓存或任务队列
```

## 21. 总结

V2 技术架构保持轻量：

```text
复习计划：新增 review_tasks + review_service
学习统计：优先聚合现有表
闪卡：新增 flashcards，可后置
批量音频：复用 tts_service 和 audio_files
AI 增强：新增 text_splitter 和 dedup_service
导入导出：优先 CSV StreamingResponse
```

推荐实现顺序：

```text
1. 首页统计增强
2. 复习计划
3. 批量音频
4. AI 分段和去重
5. 闪卡
6. 导入导出
```
