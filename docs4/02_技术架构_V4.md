# AI Study Cast - V4 技术架构

## 1. 架构原则

V4 延续 V3 架构：

```text
前端：Vue 3 + Vite + TypeScript + Element Plus + Vue Router + Pinia + Axios
后端：FastAPI + SQLAlchemy + SQLite + Pydantic + httpx
AI：OpenAI-compatible Chat Completions
TTS：mock provider + XiaomiTTSProvider 预留
```

原则：

```text
1. 不重写项目。
2. 后端封装所有 AI / TTS 调用。
3. 数据库仍使用 SQLite。
4. 路由保持轻薄，业务逻辑放 services。
5. 继续通过 core/paths.py 管理本地路径。
6. 继续使用 Base.metadata.create_all(bind=engine)。
7. 暂不引入 Alembic。
```

## 4. 数据流

AI 调用数据流：

```text
前端请求 → 后端路由 → AI service → AI provider → JSON parser → 业务保存
                         ↓
                    AI log service
```

要求：

```text
1. AI log service 记录调用结果。
2. 业务保存只在解析成功且数据有效时执行。
3. 失败时只保存失败日志，不保存空业务数据。
```

复习数据流：

```text
答题提交 → 判题 → 保存答题记录 → 更新错题 → review_service 更新知识点复习状态
```

音频数据流：

```text
选择知识点 → TTS service → 保存音频文件 → audio_files → 播放列表展示
```

## 5. 错误响应规范

继续使用 V3 统一错误响应。

V4 新增错误场景：

```text
AI 日志不存在
复习计划计算失败
没有待复习知识点
题型不支持
PDF 文本提取失败
扫描 PDF 暂不支持
CSV 导出失败
批量音频生成失败
播放列表为空
```

前端必须显示中文错误提示，不能只显示：

```text
error
failed
undefined
```

---

## 6. 安全与隐私

必须遵守：

```text
1. API Key 不写入代码。
2. API Key 不写入 AI 日志。
3. API Key 不返回前端。
4. AI 输入输出日志只保存摘要，避免过大。
5. 导出文件不包含密钥和配置。
6. 本地备份仍不额外加密。
```

---

## 7. 兼容性要求

V4 必须兼容 V1/V2/V3 已有数据。

字段新增策略：

```text
1. 新字段必须有默认值或允许为空。
2. 旧数据不能因缺少 V4 字段而启动失败。
3. create_all 不会自动修改已有表时，需要提供安全补字段逻辑。
4. 补字段逻辑必须幂等。
```
