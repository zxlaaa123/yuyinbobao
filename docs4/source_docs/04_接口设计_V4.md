# 04_接口设计_V4

# AI Study Cast - V4 接口设计

## 1. 接口原则

所有业务接口仍以 `/api` 开头。

音频静态文件仍使用：

```text
/audio/xxx.mp3
```

接口返回继续遵守 V3 错误响应规范。

---

## 2. AI 日志接口

### GET /api/ai/logs

用途：查询 AI 调用日志。

查询参数：

```text
status
call_type
page
page_size
```

返回字段：

```text
items
total
page
page_size
```

### GET /api/ai/logs/{id}

用途：查看单条 AI 日志详情。

返回：

```text
id
call_type
model_name
base_url
status
error_type
error_message
input_summary
output_summary
json_parse_status
token 用量
duration_ms
created_at
```

### DELETE /api/ai/logs/{id}

用途：删除单条 AI 日志。

要求：

```text
前端必须二次确认。
```

---

## 3. 复习接口

### GET /api/review/today

用途：获取今日复习概览和知识点列表。

返回：

```text
due_count
overdue_count
weak_count
items
```

### POST /api/review/submit-result

用途：提交知识点复习结果。

请求：

```text
knowledge_point_id
is_correct
question_id
```

返回：

```text
knowledge_point_id
review_status
mastery_level
next_review_at
```

---

## 4. 题目接口增强

### POST /api/questions/generate

用途：AI 生成题目。

请求：

```text
knowledge_base_id
knowledge_point_ids
question_types
difficulty
count
```

question_types 支持：

```text
single_choice
multiple_choice
true_false
fill_blank
short_answer
```

返回：

```text
created_count
items
```

---

## 5. 练习会话接口

### GET /api/practice/sessions

用途：查询练习会话历史。

查询参数：

```text
page
page_size
mode
```

### GET /api/practice/sessions/{id}

用途：查询练习会话详情。

### POST /api/practice/sessions

用途：创建或提交练习会话。

请求：

```text
mode
items
duration_seconds
```

items：

```text
question_id
knowledge_point_id
user_answer
is_correct
duration_seconds
```

---

## 6. PDF 导入接口

### POST /api/materials/import-pdf

用途：导入文本型 PDF。

请求：

```text
multipart/form-data
knowledge_base_id
file
```

返回：

```text
material_id
title
content_length
```

错误：

```text
PDF 文本提取失败
扫描 PDF 暂不支持
未选择知识库
```

---

## 7. 导出接口

### GET /api/export/questions.csv

用途：导出题目 CSV。

查询参数：

```text
scope
knowledge_base_id
```

scope：

```text
all
knowledge_base
wrong_questions
weak_points
```

CSV 字段：

```text
知识点
问题
答案
解析
知识库名称
题型
难度
```

---

## 8. 音频接口增强

### POST /api/tts/batch-generate

用途：批量生成知识点音频。

请求：

```text
knowledge_point_ids
provider
```

### GET /api/audio/playlists

查询参数：

```text
knowledge_base_id
review_only
listened
```

### PATCH /api/audio/{id}/listened

请求：

```text
listened
```
