# 05_AI与提示词设计_V4

# AI Study Cast - V4 AI 与提示词设计

## 1. AI 原则

V4 仍使用 OpenAI-compatible Chat Completions。

必须遵守：

```text
1. 所有 AI 调用走后端。
2. 前端不保存、不读取、不传递 API Key。
3. AI 输出尽量要求 JSON。
4. 后端必须容错解析 JSON。
5. AI 失败时只记录日志，不保存脏业务数据。
6. AI 请求必须设置 timeout。
```

---

## 2. AI 日志

每次 AI 调用必须记录：

```text
call_type
model_name
base_url
status
duration_ms
error_type
error_message
input_summary
output_summary
json_parse_status
token 用量
created_at
```

call_type：

```text
extract_points
generate_questions
test_connection
practice_summary
```

安全要求：

```text
1. 不记录 API Key。
2. 错误信息中过滤疑似密钥。
3. input_summary 和 output_summary 限制长度。
4. 不把完整用户资料内容写入日志。
```

---

## 3. 多题型生成提示词

题型：

```text
single_choice
multiple_choice
true_false
fill_blank
short_answer
```

AI 输出 JSON 格式：

```json
{
  "questions": [
    {
      "question_type": "single_choice",
      "difficulty": "medium",
      "question": "题干",
      "options": ["A", "B", "C", "D"],
      "answer": "A",
      "reference_answer": "参考答案",
      "explanation": "解析",
      "knowledge_point_title": "知识点标题"
    }
  ]
}
```

要求：

```text
1. single_choice 必须只有一个正确答案。
2. multiple_choice 可以有多个正确答案。
3. true_false 答案只能是 true 或 false。
4. fill_blank 必须提供参考答案。
5. short_answer 只提供参考答案，不自动判分。
6. 每道题必须绑定知识点。
```

---

## 4. 会话总结提示词

输入：

```text
练习题目
用户答案
正确答案
知识点
是否答对
```

输出 JSON：

```json
{
  "summary": "本次练习总结",
  "weak_points": [
    {
      "knowledge_point_id": 1,
      "reason": "薄弱原因"
    }
  ],
  "suggestion": "下一步复习建议"
}
```

失败处理：

```text
AI 总结失败时仍保存本地统计。
summary 使用默认中文说明。
失败原因写入 AI 日志。
```

---

## 5. JSON 解析兼容

必须支持：

```text
纯 JSON
Markdown JSON 代码块
前后带额外文本的 JSON
```

解析失败：

```text
1. 记录 AI 日志 json_parse_status = failed。
2. 返回中文错误。
3. 不保存空题目或空知识点。
```
