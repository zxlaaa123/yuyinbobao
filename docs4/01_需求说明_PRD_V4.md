# 01_需求说明_PRD_V4

# AI Study Cast - V4 需求说明 PRD

## 1. 文档目的

本文档定义 V4 的产品目标、功能范围、成功标准和不做内容。

V4 不推翻 V3，而是在 V3 基础上补齐长期学习闭环。

---

## 2. V4 产品目标

### 2.1 必做目标

```text
1. 增强 AI 调用日志，便于排查失败和观察调用质量。
2. 建立间隔重复复习机制。
3. 支持多题型题目生成和刷题。
4. 支持练习会话统计和总结。
5. 支持文本型 PDF 导入。
6. 支持题目 CSV 导出。
7. 增强音频复习播放体验。
```

### 2.2 推荐目标

```text
1. 首页展示今日待复习、逾期复习、薄弱知识点。
2. AI 日志支持按调用类型和状态筛选。
3. 练习结束后推荐薄弱知识点复习。
4. 音频支持播放列表、倍速、已听状态。
5. 设置页展示 AI/TTS 连接测试和配置状态。
```

### 2.3 可选目标

```text
1. CSV 导出兼容 Anki 导入格式。
2. 批量生成复习音频。
3. AI 输出质量评分字段。
4. 复习计划手动重置。
5. 知识点熟练度手动调整。
```

---

## 3. 用户角色

V4 仍只有一个角色：

```text
本机学习者
```

用户目标：

```text
1. 知道今天该复习什么。
2. 通过多题型检测掌握情况。
3. 看到薄弱知识点和错题。
4. 排查 AI 失败原因。
5. 把题目导出到外部工具。
6. 用音频进行碎片化复习。
```

---

## 4. V4 不做内容

```text
账号系统
多用户
云同步
团队协作
复杂权限
Docker
Redis / Celery
复杂 RAG
OCR
Office 深度解析
移动端 App
桌面端打包
简答题 AI 自动判分
Anki .apkg 完整生成
```

---

## 5. V4 核心功能

## 5.1 AI 日志增强

记录：

```text
调用类型
模型名称
base_url
请求时间
耗时
状态
错误类型
错误信息
输入摘要
输出摘要
JSON 解析状态
token 用量
```

要求：

```text
1. 不保存 API Key。
2. 不向前端返回 API Key。
3. 不打印完整密钥。
4. AI 失败日志不污染业务数据。
5. 日志页面能筛选、查看、复制错误信息。
```

## 5.2 间隔重复复习

知识点增加复习状态：

```text
熟练度
复习次数
连续答对次数
连续答错次数
上次复习时间
下次复习时间
复习状态
```

答题影响：

```text
答对：延后复习
答错：提前复习并进入错题
连续答对：提高熟练度
连续答错：降低熟练度
```

## 5.3 多题型生成

支持题型：

```text
single_choice
multiple_choice
true_false
fill_blank
short_answer
```

简答题：

```text
V4 只展示参考答案，不做 AI 自动判分。
```

## 5.4 学习会话

一次练习结束后保存：

```text
总题数
正确数
错误数
正确率
耗时
涉及知识点
薄弱知识点
错题列表
建议复习内容
```

## 5.5 PDF 文本导入

支持：

```text
文本型 PDF
TXT
Markdown
手动粘贴
```

不支持：

```text
扫描件 OCR
图片公式识别
复杂版面还原
```

## 5.6 CSV 导出

导出范围：

```text
当前知识库
错题
全部题目
薄弱知识点
```

导出字段：

```text
知识点
问题
答案
解析
知识库名称
题型
难度
```

## 5.7 音频复习增强

新增：

```text
播放列表
按知识库筛选
按复习计划筛选
倍速播放
已听状态
批量生成音频
重新生成音频
```

---

## 6. 成功标准

V4 完成后必须满足：

```text
1. AI 成功和失败调用都能进入日志。
2. API Key 不会进入日志、前端响应或文档。
3. 知识点能根据答题结果更新复习状态。
4. 首页能看到今日待复习和薄弱知识点。
5. 五种题型都能生成、保存、展示。
6. 练习结束能保存会话统计。
7. 文本型 PDF 能导入为资料。
8. CSV 能导出并用表格软件打开。
9. 音频页能按列表播放并标记已听。
10. V1/V2/V3 核心流程不被破坏。
```
