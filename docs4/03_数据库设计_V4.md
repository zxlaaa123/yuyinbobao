# 03_数据库设计_V4

# AI Study Cast - V4 数据库设计

## 1. 数据库原则

数据库仍使用：

```text
SQLite
data/app.db
SQLAlchemy
Base.metadata.create_all(bind=engine)
```

V4 暂不引入：

```text
Alembic
Redis
Celery
多租户
```

已有表必须兼容 V1/V2/V3 数据。

---

## 2. 新增表：ai_call_logs

用途：记录 AI 调用成功和失败信息。

字段建议：

```text
id
call_type
model_name
base_url
status
error_type
error_message
input_summary
output_summary
json_parse_status
prompt_tokens
completion_tokens
total_tokens
duration_ms
created_at
```

安全规则：

```text
1. 不保存 API Key。
2. 不保存完整请求头。
3. 输入和输出只保存摘要。
4. 错误信息中必须过滤密钥。
```

---

## 3. 新增表：practice_sessions

用途：记录一次练习会话。

字段建议：

```text
id
title
mode
total_count
correct_count
wrong_count
accuracy
duration_seconds
weak_knowledge_point_ids
suggestion
started_at
ended_at
created_at
```

mode 示例：

```text
normal
review
wrong_questions
weak_points
```

---

## 4. 新增表：practice_session_items

用途：记录练习会话中的每道题。

字段建议：

```text
id
session_id
question_id
knowledge_point_id
user_answer
is_correct
duration_seconds
created_at
```

关联：

```text
practice_sessions.id → practice_session_items.session_id
questions.id → practice_session_items.question_id
knowledge_points.id → practice_session_items.knowledge_point_id
```

---

## 5. knowledge_points 增强字段

新增字段：

```text
mastery_level
review_count
correct_streak
wrong_streak
last_reviewed_at
next_review_at
review_status
```

默认值：

```text
mastery_level = 0
review_count = 0
correct_streak = 0
wrong_streak = 0
review_status = new
next_review_at = created_at
```

review_status 取值：

```text
new
learning
review
mastered
```

---

## 6. questions 增强字段

新增字段：

```text
question_type
difficulty
reference_answer
```

question_type 取值：

```text
single_choice
multiple_choice
true_false
fill_blank
short_answer
```

difficulty 取值：

```text
easy
medium
hard
```

兼容规则：

```text
旧题目默认 question_type = single_choice
旧题目默认 difficulty = medium
```

---

## 7. audio_files 增强字段

新增字段：

```text
duration_seconds
listened
listened_at
playlist_name
playback_source
```

默认值：

```text
listened = false
playlist_name = null
playback_source = knowledge_point
```

---

## 8. 数据升级策略

由于 V4 不引入 Alembic，需在后端启动时执行幂等补字段。

要求：

```text
1. 检查字段是否存在。
2. 缺失时 ALTER TABLE ADD COLUMN。
3. 不删除旧字段。
4. 不重建已有表。
5. 补字段失败时启动失败并给出明确日志。
```
