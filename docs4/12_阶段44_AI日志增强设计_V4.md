## 2. V4 设计结论

阶段 45 不新建另一套日志系统。

采用：

```text
保留 ai_call_logs 表
保留 /api/ai-call-logs 现有接口
增强字段、脱敏、详情、删除、兼容别名
```

V4 文档中的字段映射：

```text
call_type → operation
model_name → model
base_url → base_url_host
input_summary → request_summary
output_summary → response_summary
```

原因：

```text
1. 现有字段已覆盖大部分 V4 需求。
2. 复用现有前端页面和路由，风险更低。
3. 避免重复建表、重复接口、重复页面。
```

## 3. 阶段 45 必须补齐的后端差距

### 3.1 数据字段

在 `ai_call_logs` 增加字段：

```text
error_type
json_parse_status
http_status_code
```

字段说明：

```text
error_type：错误类型，例如 timeout / http_error / json_parse_error / validation_error / unknown
json_parse_status：not_required / success / failed / fixed
http_status_code：外部 AI 接口 HTTP 状态码
```

默认值：

```text
error_type = null
json_parse_status = not_required
http_status_code = null
```

兼容要求：

```text
1. 通过 backend/app/core/migrations.py 做幂等补字段。
2. 不删除旧字段。
3. 不重建表。
4. 旧日志可以继续展示。
```

### 3.2 日志服务

增强 `create_ai_call_log` 参数：

```text
error_type
json_parse_status
http_status_code
```

增强 `_summary`：

```text
1. 限制长度继续保留。
2. 增加密钥脱敏。
3. 脱敏 Authorization、Bearer、api_key、secret、password。
```

日志失败处理继续保持：

```text
日志写入失败不能中断 AI 主流程。
```

### 3.3 AI 调用埋点

增强 `AIService`：

```text
1. HTTPStatusError 记录 http_status_code 和 error_type=http_error。
2. TimeoutException 记录 error_type=timeout。
3. JSON 解析失败记录 json_parse_status=failed。
4. JSON 修复成功记录 json_parse_status=fixed。
5. 业务校验失败记录 error_type=validation_error。
6. 成功解析记录 json_parse_status=success。
```

需要覆盖：

```text
extract_knowledge_points
generate_questions
fix_json
generate_flashcards
test_ai_connection
```

注意：

```text
settings.py 里的 /api/settings/test-ai 当前直接用 httpx 调用，需要阶段 45 纳入日志。
```

### 3.4 后端接口

新增：

建议阶段 45 先做现有前缀，别名可在阶段 46 前补。

## 5. 脱敏规则

日志入库前必须脱敏。

替换规则：

```text
Authorization: Bearer xxx → Authorization: Bearer [REDACTED]
Bearer xxx → Bearer [REDACTED]
api_key=xxx → api_key=[REDACTED]
"api_key": "xxx" → "api_key": "[REDACTED]"
secret=xxx → secret=[REDACTED]
password=xxx → password=[REDACTED]
```

脱敏范围：

```text
request_summary
response_summary
error_message
```

## 6. 阶段 45 验收标准

必须通过：

```text
1. AI 成功调用写入日志。
2. AI HTTP 失败写入 error_type=http_error。
3. AI 超时写入 error_type=timeout。
4. JSON 解析失败写入 json_parse_status=failed。
5. JSON 修复成功写入 json_parse_status=fixed。
6. 业务校验失败写入 error_type=validation_error。
7. GET /api/ai-call-logs/{id} 可查看详情。
8. DELETE /api/ai-call-logs/{id} 可删除日志。
9. 日志内容不包含 API Key。
10. 旧日志仍能在页面展示。
```
