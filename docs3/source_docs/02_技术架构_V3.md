# 02_技术架构_V3

# AI Study Cast - V3 技术架构

## 1. 架构原则

V3 继续使用本地单机架构。

```text
前端 Vue 3
后端 FastAPI
数据库 SQLite
文件 data/
AI/TTS 只走后端
```

不引入新基础设施。

---

## 2. 前端架构调整

目标结构：

```text
frontend/src/
├─ api/
├─ components/
├─ layouts/
│  └─ MainLayout.vue
├─ router/
│  └─ index.ts
├─ stores/
│  ├─ app.ts
│  └─ settings.ts
├─ types/
│  ├─ api.ts
│  ├─ knowledge.ts
│  ├─ practice.ts
│  └─ settings.ts
├─ utils/
├─ views/
├─ App.vue
└─ main.ts
```

迁移原则：

```text
1. 先搬路由，不改页面行为
2. 再抽布局，不改样式大结构
3. 再搬共享类型
4. stores 只放跨页面状态，不强行重构全部页面
```

---

## 3. 后端架构调整

目标：

```text
路由负责参数和响应
services 负责业务
models 只定义表
schemas 负责请求/响应结构
core 负责配置、路径、数据库
utils 负责通用工具
```

建议新增：

```text
backend/app/services/backup_service.py
backend/app/services/search_service.py
backend/app/services/activity_log_service.py
backend/app/schemas/common.py
backend/app/utils/errors.py
```

---

## 4. 错误响应规范

V3 推荐统一错误格式：

```json
{
  "detail": "中文错误说明",
  "code": "ERROR_CODE",
  "hint": "可选操作建议"
}
```

前端展示：

```text
优先 detail
其次 message
最后显示“操作失败，请稍后重试”
```

---

## 5. 数据安全

V3 新增备份恢复。

备份包建议：

```text
backup_YYYYMMDD_HHMMSS.zip
├─ app.db
├─ audio/
├─ uploads/
└─ manifest.json
```

恢复规则：

```text
1. 恢复前必须二次确认
2. 恢复前自动创建当前数据备份
3. 只允许恢复到 data/ 目录
4. 不恢复 .env
```

---

## 6. AI/TTS 可观测性

AI 调用日志：

```text
ai_call_logs
```

TTS 可继续使用：

```text
audio_files.status
audio_files.error_message
```

必要时新增：

```text
audio_files.audio_type
audio_files.provider
audio_files.format
audio_files.duration
```

---

## 7. 兼容性要求

V3 必须兼容已有 `data/app.db`。

规则：

```text
1. 新表可用 Base.metadata.create_all 创建
2. 给旧表新增字段时必须考虑 SQLite 旧库
3. 暂不引入 Alembic，除非用户明确要求
4. 如需新增字段，提供兼容初始化逻辑
```
