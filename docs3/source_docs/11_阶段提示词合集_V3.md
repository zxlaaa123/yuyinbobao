# 11_阶段提示词合集_V3

# AI Study Cast - V3 阶段提示词合集

## 使用说明

每次只复制一个阶段。

不要让 AI 一次做完整 V3。

---

## 阶段 31

```text
请开始阶段 31：V2 最终验收和问题收口。
要求：
1. 先阅读 docs2/10_验收清单_V2.md 和 docs3/08_开发阶段计划_V3.md。
2. 只做验收和最小 bug 修复。
3. 不新增 V3 功能。
4. 完成后输出通过项、失败项、修改文件、测试命令。
```

## 阶段 32

```text
请开始阶段 32：前端结构整理。
要求：
1. 新增 frontend/src/router/index.ts。
2. 新增 frontend/src/layouts/MainLayout.vue。
3. main.ts 只保留 app 初始化。
4. App.vue 简化为 router-view。
5. 不改变页面行为和路由路径。
6. 完成后运行 npm run build。
```

## 阶段 33

```text
请开始阶段 33：后端错误处理和响应规范。
要求：
1. 统一核心接口错误提示。
2. 前端新增错误信息提取工具。
3. 先改高频页面，不全量重构。
4. 错误提示必须是中文。
5. 完成后说明如何测试空标题、AI Key 未配置、TTS 失败。
```

## 阶段 34

```text
请开始阶段 34：本地备份与恢复。
要求：
1. 新增 backup_records 表。
2. 新增 backup_service.py。
3. 新增 /api/backups 接口。
4. 前端提供创建、恢复、删除备份入口。
5. 恢复前必须二次确认。
6. 不备份 .env。
```

## 阶段 35

```text
请开始阶段 35：学习会话和练习记录增强。
要求：
1. 新增 study_sessions 表。
2. 练习开始创建 session。
3. 练习结束保存题目数、正确数、正确率。
4. 首页展示最近学习记录。
5. 不破坏原刷题流程。
```

## 阶段 36

```text
请开始阶段 36：复习计划增强。
要求：
1. 区分今日、逾期、稍后、已完成。
2. 支持复习质量反馈。
3. 简单计算下次复习时间。
4. 不引入复杂 FSRS 库。
5. 不破坏现有 review 接口。
```

## 阶段 37

```text
请开始阶段 37：导入能力增强。
要求：
1. 支持 TXT 和 Markdown 文件上传。
2. 文件保存到 data/uploads。
3. 解析为资料正文。
4. 前端资料导入页增加上传入口。
5. 不做 PDF、Word、OCR。
```

## 阶段 38

```text
请开始阶段 38：AI 调用日志和成本估算。
要求：
1. 新增 ai_call_logs 表。
2. AIService 记录成功和失败日志。
3. 不记录 API Key。
4. 默认只保存摘要和字符数。
5. 前端提供日志查看入口。
```

## 阶段 39

```text
请开始阶段 39：TTS 音频体验增强。
要求：
1. 完善音频类型和格式展示。
2. 支持失败音频重新生成。
3. 删除音频时删除本地文件。
4. 不打印 TTS API Key。
5. mock provider 仍可用。
```

## 阶段 40

```text
请开始阶段 40：本地搜索增强。
要求：
1. 新增 /api/search。
2. 搜索知识点标题、摘要、详情、标签和资料标题。
3. 前端增加搜索入口。
4. 点击结果可跳转。
5. 初期用 SQLite LIKE，不做复杂 RAG。
```

## 阶段 41

```text
请开始阶段 41：界面体验统一。
要求：
1. 统一空状态。
2. 统一删除确认文案。
3. 统一 loading。
4. 统一错误提示。
5. 检查移动端基本布局。
6. 不新增业务功能。
```

## 阶段 42

```text
请开始阶段 42：V3 整体联调和最终验收。
要求：
1. 按 docs3/10_验收清单_V3.md 验收。
2. 先跑 V1 主流程，再跑 V2，再跑 V3。
3. 发现问题只做最小修复。
4. 最后输出最终验收报告。
```
