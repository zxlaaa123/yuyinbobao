# 04_接口设计_V3

# AI Study Cast - V3 接口设计

## 1. 接口原则

继续遵守：

```text
所有业务接口以 /api 开头
音频静态文件不以 /api 开头
前端不直连 AI / TTS 外部 API
错误提示使用中文
删除操作需要前端二次确认
```

---

## 2. 备份恢复接口

```http
GET /api/backups
POST /api/backups/create
POST /api/backups/restore
DELETE /api/backups/{backup_id}
```

### POST /api/backups/create

请求：

```json
{
  "note": "手动备份"
}
```

响应：

```json
{
  "id": 1,
  "filename": "backup_20260509_120000.zip",
  "status": "success"
}
```

### POST /api/backups/restore

请求：

```json
{
  "backup_id": 1,
  "confirm": true
}
```

要求：

```text
confirm 必须为 true
恢复前自动备份当前数据
```

---

## 3. 学习会话接口

```http
POST /api/study-sessions
POST /api/study-sessions/{session_id}/finish
GET /api/study-sessions
GET /api/study-sessions/{session_id}
```

### POST /api/study-sessions

请求：

```json
{
  "knowledge_base_id": 1,
  "knowledge_point_id": null
}
```

### POST /api/study-sessions/{session_id}/finish

请求：

```json
{
  "question_count": 10,
  "correct_count": 8
}
```

---

## 4. AI 调用日志接口

```http
GET /api/ai/logs
GET /api/ai/logs/{log_id}
DELETE /api/ai/logs/{log_id}
```

筛选参数：

```text
call_type
status
model
date_from
date_to
limit
offset
```

---

## 5. 搜索接口

```http
GET /api/search
```

参数：

```text
q
scope=all|knowledge_points|materials
knowledge_base_id
limit
```

响应：

```json
{
  "items": [
    {
      "type": "knowledge_point",
      "id": 1,
      "title": "标题",
      "summary": "摘要",
      "url": "/knowledge-points/1"
    }
  ]
}
```

---

## 6. 音频增强接口

可扩展：

```http
POST /api/audio-files/{audio_id}/regenerate
GET /api/audio-files/{audio_id}/metadata
```

保持已有接口不变：

```http
GET /api/audio-files
DELETE /api/audio-files/{audio_id}
POST /api/tts/generate
POST /api/tts/generate-batch
```

---

## 7. 错误响应建议

```json
{
  "detail": "资料正文不能为空",
  "code": "MATERIAL_CONTENT_EMPTY",
  "hint": "请填写正文后再保存"
}
```

前端必须兼容旧格式：

```json
{
  "detail": "错误信息"
}
```
