# README_使用顺序_V3

# AI Study Cast - V3 文档使用顺序

## 1. V3 定位

V3 建立在 V1 + V2 已完成基础上。

V3 不是重写项目。

V3 的目标：

```text
从“功能可用”
升级为
“稳定、好维护、复习闭环更完整、数据更安全”
```

V3 优先解决：

```text
1. V2 最终验收和遗留 bug
2. 前端结构规范化
3. 后端服务边界更清晰
4. 本地备份和恢复
5. 学习复习体验增强
6. AI / TTS 调用可观测和可控
```

---

## 2. V3 前置条件

进入 V3 前必须完成：

```text
1. docs2/10_验收清单_V2.md 完整跑通
2. 前端 npm run build 通过
3. 后端 app.main 可导入
4. /api/health 可访问
5. 主流程可跑通：
   导入资料 → 提取知识点 → 生成题目 → 刷题 → 错题 → 生成音频 → 播放
```

如果 V2 验收不通过：

```text
先修 V2，不进入 V3。
```

---

## 3. V3 继续禁止

V3 仍然不做：

```text
登录注册
多用户权限
云同步
Docker
Redis
Celery
消息队列
微服务
手机 App
浏览器插件
付费系统
复杂 RAG
前端直接调用 AI / TTS 外部 API
把 API Key 写进代码
```

```text
不破坏 V1/V2 > 能运行 > 数据安全 > 错误可理解 > 复习更顺 > AI/TTS 可控 > 搜索更快 > 界面更统一
```
