# 01_需求说明_PRD_V3

# AI Study Cast - V3 需求说明 PRD

## 1. 文档目的

本文档用于规划 V3。

V3 不推翻 V1/V2。

V3 聚焦：

```text
稳定性
可维护性
数据安全
学习闭环
AI/TTS 可控性
本地搜索
体验一致性
```

---

## 2. V3 产品目标

### 2.1 必做目标

```text
1. 完成 V2 最终验收
2. 修复 V2 遗留 bug
3. 规范前端目录：router / layouts / types / stores
4. 统一后端错误响应和前端错误提示
5. 支持本地数据备份与恢复
```

### 2.2 推荐目标

```text
1. 增强练习记录，支持一次练习会话统计
2. 增强复习计划，支持更清晰的待复习状态
3. 支持 Markdown / TXT 文件导入
4. 记录 AI 调用日志，便于排查失败和估算消耗
5. 增强 TTS 音频元数据和播放体验
6. 支持知识点本地搜索
```

### 2.3 可选目标

```text
1. 简化版间隔重复算法
2. 知识库归档
3. 学习目标和每日计划
4. 导出备份包
5. Anki 兼容导出
```

---

## 3. 用户角色

V3 仍只有一个角色：

```text
本机学习者
```

用户目标：

```text
1. 安心保存学习数据
2. 快速找到知识点
3. 继续练习和复习
4. 了解学习进展
5. 控制 AI / TTS 调用失败风险
```

---

## 4. V3 不做内容

```text
账号系统
多用户
云端同步
团队协作
复杂权限
Docker
Redis / Celery
复杂向量 RAG
OCR
Office 深度解析
移动端 App
桌面端打包
```

---

## 5. V3 核心功能

## 5.1 工程结构规范化

前端：

```text
main.ts 只负责创建 app
router/index.ts 管理路由
layouts/ 管理布局
types/ 管理共享类型
stores/ 管理轻量状态
```

后端：

```text
路由保持轻薄
复杂逻辑放 services/
错误响应格式统一
文件路径仍由 core/paths.py 管理
```

## 5.2 本地备份与恢复

支持：

```text
1. 创建备份包
2. 下载备份包
3. 从备份包恢复
4. 恢复前二次确认
```

备份内容：

```text
data/app.db
data/audio/
data/uploads/
必要元信息
```

## 5.3 学习会话

新增学习会话概念：

```text
一次刷题开始到结束为一个 session
记录开始时间、结束时间、题目数、正确数、知识库、知识点
```

用于：

```text
1. 首页展示最近学习
2. 统计练习效率
3. 复习计划参考
```

## 5.4 复习计划增强

增强内容：

```text
1. 今日待复习
2. 已逾期
3. 稍后复习
4. 已完成
5. 复习来源说明
```

初期不做完整 FSRS。

可采用简单规则：

```text
错题：1 天、3 天、7 天
高重要知识点：2 天、5 天、14 天
普通知识点：3 天、7 天、21 天
```

## 5.5 AI 调用日志

记录：

```text
调用类型
模型
输入摘要
输出摘要
状态
错误信息
耗时
创建时间
```

不记录完整 API Key。

默认不保存完整原文，避免数据库膨胀。

## 5.6 TTS 音频体验

增强：

```text
1. 音频时长记录
2. 音频类型更明确
3. 失败原因更清楚
4. 重新生成音频
5. 删除音频同时删除本地文件
```

## 5.7 本地搜索

先做 SQLite LIKE 搜索。

可选做 SQLite FTS5。

搜索范围：

```text
知识点标题
知识点摘要
知识点详情
标签
资料标题
```

---

## 6. 成功标准

V3 完成后：

```text
1. V1/V2 主流程不回归
2. 数据可备份恢复
3. 前端结构更清晰
4. 错误提示更一致
5. 练习和复习记录更完整
6. AI/TTS 问题更容易排查
```
