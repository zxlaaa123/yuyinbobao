# 03_数据库设计_V3

# AI Study Cast - V3 数据库设计

## 1. 数据库原则

继续使用：

```text
SQLite
data/app.db
SQLAlchemy
Base.metadata.create_all
```

V3 不引入 Alembic。

如需改旧表字段，必须提供兼容逻辑。

---

## 2. 建议新增表

## 2.1 study_sessions

用途：

```text
记录一次刷题会话。
```

字段：

```text
id
knowledge_base_id nullable
knowledge_point_id nullable
started_at
ended_at nullable
question_count
correct_count
accuracy
duration_seconds nullable
created_at
updated_at
```

## 2.2 ai_call_logs

用途：

```text
记录 AI 调用状态，便于排错。
```

字段：

```text
id
call_type
provider
model
status
input_chars
output_chars
duration_ms
error_message nullable
request_summary nullable
response_summary nullable
created_at
```

禁止保存：

```text
API Key
完整敏感原文
```

## 2.3 backup_records

用途：

```text
记录备份文件。
```

字段：

```text
id
filename
file_path
file_size
status
note nullable
created_at
```

## 2.4 search_index_records 可选

如果启用 FTS5，可新增虚拟表。

初期可不建表，先用 LIKE 搜索。

## 3. 旧表建议增强

## 3.1 audio_files

建议新增字段：

```text
audio_type
provider
format
duration
source_count
```

兼容规则：

```text
旧数据缺字段时前端按默认值展示。
```

## 3.2 review_tasks

建议新增字段：

```text
due_at
last_reviewed_at
review_count
quality_score
next_interval_days
```

初期可继续使用现有字段。

---

## 4. 删除和级联规则

V3 必须明确清理关联数据。

删除资料：

```text
materials
关联 knowledge_points
关联 questions
关联 answer_records
关联 wrong_questions
关联 audio_files
本地音频文件
```

删除知识库：

```text
V3 默认仍建议非空禁止删除。
```

删除音频：

```text
删除数据库记录
尝试删除本地文件
文件不存在不报致命错误
```

---

## 5. 数据迁移策略

不使用 Alembic 时：

```text
1. 新表直接 create_all
2. 新字段通过兼容函数检查 PRAGMA table_info
3. 缺字段时 ALTER TABLE ADD COLUMN
4. 每次迁移必须可重复执行
```

建议文件：

```text
backend/app/core/migrations.py
```
