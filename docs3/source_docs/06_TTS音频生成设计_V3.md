# 06_TTS音频生成设计_V3

# AI Study Cast - V3 TTS 音频生成设计

## 1. V3 TTS 目标

V3 不重写 TTS。

重点增强：

```text
格式一致
状态清楚
可重新生成
播放体验更好
错误更容易排查
```

---

## 2. Provider

继续支持：

```text
mock
xiaomi
```

mock 用于本地流程验证。

xiaomi 只从后端调用。

---

## 3. 音频格式

支持：

```text
mp3
wav
pcm16
```

规则：

```text
1. 文件扩展名必须和实际内容一致
2. 默认格式 mp3
3. 不支持的格式回退 mp3
4. 前端只播放浏览器支持的格式
```

---

## 4. 音频类型

建议记录：

```text
single_point
batch_points
daily_review
wrong_questions
```

用于前端筛选。

---

## 5. 重新生成

新增操作：

```text
重新生成音频
```

规则：

```text
1. 保留旧记录或覆盖旧文件，两者择一
2. V3 初期建议新建记录，旧记录可删除
3. 失败时保留错误信息
```

---

## 6. 播放体验

前端增强：

```text
1. 最近播放
2. 播放进度
3. 生成失败重试
4. 按音频类型筛选
5. 没有音频时显示中文空状态
```

---

## 7. 安全规则

禁止：

```text
日志打印 API Key
前端保存 TTS Key
把真实 TTS Key 写进文档或代码
```
