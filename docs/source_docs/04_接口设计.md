# 04_接口设计

# AI 知识点学习与音频播报系统 - 接口设计

## 1. 文档目的

本文档用于定义 V1 版本前后端接口。

它主要回答：

```text
前端应该调用哪些接口？
每个接口的路径是什么？
请求参数怎么传？
返回数据长什么样？
错误时怎么返回？
AI 和 TTS 接口如何封装？
```

本项目采用前后端分离架构：

```text
前端：Vue3 + Vite
后端：FastAPI
通信方式：HTTP JSON API
```

---

## 2. 接口基础约定

## 2.1 后端基础地址

默认后端地址：

```text
http://localhost:8000
```

---

## 2.2 API 前缀

所有业务接口统一使用：

```text
/api
```

示例：

```text
http://localhost:8000/api/health
```

---

## 2.3 音频静态文件地址

音频文件不走 `/api`，使用静态文件路径：

```text
/audio
```

示例：

```text
http://localhost:8000/audio/kp_1_20260506_153000.mp3
```

---

## 2.4 请求格式

默认使用：

```http
Content-Type: application/json
```

请求体使用 JSON。

V1 暂不做文件上传接口，后续支持 PDF / Word 时再增加 `multipart/form-data`。

---

## 2.5 返回格式

V1 推荐直接返回数据对象。

成功示例：

```json
{
  "id": 1,
  "name": "三支一扶"
}
```

列表示例：

```json
[
  {
    "id": 1,
    "name": "三支一扶"
  }
]
```

错误示例使用 FastAPI 默认格式：

```json
{
  "detail": "错误说明"
}
```

---

## 2.6 时间格式

时间字段统一使用 ISO 格式字符串：

```text
2026-05-06T15:30:00
```

字段名：

```text
created_at
updated_at
answered_at
last_wrong_at
```

---

## 2.7 分页约定

V1 可以先不做复杂分页。

但列表接口建议预留分页参数：

```text
page
page_size
```

示例：

```http
GET /api/knowledge-points?page=1&page_size=20
```

V1 也可以直接返回数组，后续再扩展分页。

推荐 V1 简化：

```text
列表接口先返回数组
```

---

## 2.8 命名约定

接口路径使用小写加中划线：

```text
knowledge-bases
knowledge-points
wrong-questions
audio-files
```

JSON 字段使用下划线：

```text
knowledge_base_id
created_at
file_url
```

## 6.6 保存并提取知识点

### 请求

```http
POST /api/materials/import-and-extract
```

### 说明

用于资料导入页面的一键操作。

流程：

```text
保存资料
调用 AI 提取知识点

返回提取结果
```

### 请求体

```json
{
  "knowledge_base_id": 1,
  "title": "政府社会职能",
  "content": "这里是学习资料正文...",
  "source": "手动粘贴",
  "note": ""
}
```

### 返回示例

```json
{
  "material": {
    "id": 1,
    "title": "政府社会职能"
  },
  "created_count": 3,
  "knowledge_points": [
    {
      "id": 1,
      "title": "政府社会职能",
      "summary": "政府在社会公共事务管理中承担的职能。",
      "importance": "high",
      "tags": ["公基", "政府职能"]
    }
  ]
}
```

### 常见错误

```json
{
  "detail": "AI API Key 未配置，请先到设置页配置"
}
```

```json
{
  "detail": "AI 返回结果不是有效 JSON，请重试"
}
```

# 10. 刷题接口

---

## 10.1 获取练习题

### 请求

```http
GET /api/practice/questions
```

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| knowledge_base_id | integer | 否 | 知识库 ID |
| knowledge_point_id | integer | 否 | 知识点 ID |
| question_type | string | 否 | 题型 |
| count | integer | 否 | 获取数量 |

### 默认值

```text
count = 10
```

### 返回示例

```json
[
  {
    "id": 1,
    "question_type": "single_choice",
    "stem": "下列哪项属于政府的社会职能？",
    "options": [
      { "key": "A", "text": "制定货币政策" },
      { "key": "B", "text": "完善社会保障体系" },
      { "key": "C", "text": "调整产业结构" },
      { "key": "D", "text": "监管市场价格" }
    ],
    "difficulty": "medium",
    "knowledge_point_id": 1,
    "knowledge_point_title": "政府社会职能"
  }
]
```

注意：

```text
练习题接口可以不返回 answer 和 analysis，避免前端提前看到答案。
```

## 10.2 提交答案

### 请求

```http
POST /api/practice/answer
```

### 请求体

```json
{
  "question_id": 1,
  "user_answer": "B"
}
```

### 返回示例：答对

```json
{
  "question_id": 1,

  "user_answer": "B",
  "correct_answer": "B",
  "analysis": "社会保障属于政府社会职能。",
  "wrong_question_id": null
}
```

```json
{
  "question_id": 1,
  "is_correct": false,
  "user_answer": "A",
  "correct_answer": "B",
  "analysis": "社会保障属于政府社会职能。",
  "wrong_question_id": 1
}
```

### 后端处理逻辑

```text

```


### 请求体

```json
{
  "knowledge_point_id": 1
}
```

### 说明

后端根据知识点内容自动拼接播报文本，然后调用小米 TTS API 生成音频。

### 返回示例：成功

```text
1. 查询知识点
2. 拼接播报文本
3. 创建 audio_files 记录，status = pending
4. 调用小米 TTS API
5. 保存 MP3 到 data/audio
6. 更新 audio_files，status = success
7. 返回 file_url
8. 如果失败，更新 status = failed 和 error_message
```

# 15. 常见错误码建议

V1 可以直接用 `detail`，但建议内部保留错误码思路。

常见错误：

```text
VALIDATION_ERROR
NOT_FOUND
DUPLICATE_NAME
AI_CONFIG_MISSING
AI_API_ERROR
AI_JSON_PARSE_ERROR
TTS_CONFIG_MISSING
TTS_API_ERROR
DATABASE_ERROR
FILE_SAVE_ERROR
```

返回示例：

```json
{
  "detail": "AI 返回结果不是有效 JSON，请重试或调整提示词"
}
```

后续可升级为：

```json
{
  "code": "AI_JSON_PARSE_ERROR",
  "message": "AI 返回结果不是有效 JSON",
  "detail": "原始错误信息"
}
```
