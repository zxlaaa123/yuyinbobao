# AI 知识点学习与音频播报系统 - 文档使用顺序

## 1. 项目定位

本项目是一个**本地运行的 AI 学习辅助 Web 应用**。

运行方式：

```text
启动后端 + 启动前端 + 浏览器访问
```

默认地址：

```text
前端：http://localhost:5173
后端：http://localhost:8000
```

核心流程：

```text
导入资料 → AI 提取知识点 → AI 生成题目 → 刷题练习 → 错题本 → TTS 生成音频 → 播放复习
```

第一版不做：

```text
登录注册
云端部署
Docker
手机 App
复杂权限
复杂 RAG
桌面打包
```

---

## 2. 推荐技术栈

前端：

```text
Vue3 + Vite + TypeScript + Element Plus + Vue Router + Pinia + Axios
```

后端：

```text
Python + FastAPI + SQLite + SQLAlchemy + Pydantic + Uvicorn
```

数据库：

```text
data/app.db
```

本地文件：

```text
data/audio/
data/uploads/
data/vector_store/
```

AI：

```text
OpenAI-compatible 接口
```

TTS：

```text
先支持 mock provider，再预留小米 TTS
```

## 5. V1 必须完成的功能

V1 只做完整学习闭环：

```text
新建知识库
粘贴学习资料
保存资料
AI 提取知识点
查看知识点
AI 生成题目
刷题练习
错题记录
TTS 生成音频
网页播放音频
设置 AI / TTS 配置
数据本地持久化
```

V1 暂不做：

```text
PDF / Word 深度解析
复杂 RAG 问答
复习计划
多用户
云同步
桌面打包
```

## 6. 推荐开发顺序

严格按阶段推进：

```text
阶段 0：初始化项目结构
阶段 1：后端最小服务
阶段 2：前端最小页面
阶段 3：前后端联通
阶段 4：数据库初始化
阶段 5：知识库管理
阶段 6：资料保存
阶段 7：AI 知识点提取
阶段 8：知识点页面
阶段 9：AI 题目生成
阶段 10：刷题练习
阶段 11：错题本
阶段 12：TTS 音频生成
阶段 13：音频播报页面
阶段 14：设置页面
阶段 15：一键启动脚本
阶段 16：整体联调
阶段 17：界面优化
阶段 18：最终验收
```

原则：

```text
每次只做一个阶段
不要一次性写完整项目
每个阶段完成后先测试
能运行后再继续下一阶段
```

## 7. 给 AI 编程工具的总提示词

可以直接复制：

```text
请先阅读 AI_RULES.md 和 docs/README_使用顺序_精简版.md。

然后按照 docs/08_开发阶段计划.md 从当前阶段开始开发。

重要规则：
1. 每次只做一个阶段。
2. 不要一次性开发完整项目。
3. 不要提前开发后续阶段。
4. 不要使用 Docker。
5. 不要做登录注册。
6. 不要把 API Key 写死到代码里。
7. 不要让前端直接调用 AI 或 TTS 外部接口。
8. 所有配置写到 .env 或 app_settings。
9. 本地数据保存到 data/。
10. 优先保证能运行，再优化界面。
11. 每次修改后说明改了哪些文件、如何启动、如何测试。

如果涉及前端页面，请参考 docs/ui-prototype.html 的视觉风格，但正式前端必须写在 frontend/ 中。
```

## 8. 每个阶段的推荐要求格式

不要只说“继续”。

建议这样说：

```text
请开始阶段 0：初始化项目结构。

要求：
1. 只做阶段 0。
2. 不要开发后续功能。
3. 修改前说明计划创建哪些文件。
4. 修改后说明完成了什么。
5. 告诉我如何检查。
```

后续阶段也按这个格式：

```text
请开始阶段 X：阶段名称。

要求：
1. 只做当前阶段。
2. 不要提前开发后续阶段。
3. 参考 docs/08_开发阶段计划.md。
4. 完成后告诉我如何启动和测试。
```

V1 完成后应能做到：

```text
启动后端

打开网页

导入资料
AI 提取知识点
查看知识点

重启后数据不丢
```

只要这条链路跑通，第一版就算成功。

---

## 10. 开发优先级

始终按这个顺序判断：

```text
能运行 > 能保存 > 能调用 AI/TTS > 能展示 > 好看 > 功能丰富
```

不要为了复杂而复杂。

先完成 V1，再考虑 V2。
