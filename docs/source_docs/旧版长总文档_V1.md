# AI 知识点学习与音频播报系统总文档

> 本文档由第一阶段 `docs/` 下分散文档整理而来，作为后续开发的主入口。旧文档仅作为历史归档参考。

## 1. 项目定位

本项目是一个本地运行的 AI 知识点学习与音频播报系统。

核心流程：

```text
导入资料 -> AI 提取知识点 -> AI 生成题目 -> 刷题 -> 错题本 -> TTS 生成音频 -> 网页播放复习
```

默认地址：

```text
后端：http://localhost:8000
前端：http://localhost:5173
```

开发优先级：

```text
能运行 > 能保存 > 能调用 > 能展示 > 好看 > 功能丰富
```

## 5. 开发规则

每次只做一个阶段。不要提前开发后续阶段。

必须遵守：

```text
不要重写整个项目
不要删除已有可运行代码
不要反复无意义修改同一个文件
不要把 API Key 写死进代码
不要让前端直接调用 AI 或 TTS 外部 API
所有 AI / TTS 调用必须走后端
优先保证能运行，再优化界面
```

修改文件前说明：


## 6. 后端设计

后端使用 FastAPI。

分层规则：

```text
API 路由：backend/app/api/routes/
数据模型：backend/app/models/
Pydantic schema：backend/app/schemas/
业务逻辑：backend/app/services/
配置：backend/app/core/config.py
数据库连接：backend/app/core/database.py
路径管理：backend/app/core/paths.py
工具函数：backend/app/utils/
```

`main.py` 只负责：

```text
创建 app
注册路由
配置 CORS
挂载静态文件
```

健康检查：

```http
GET /api/health
```

最小返回：

```json
{
  "status": "ok"
}
```

后端启动：

```bash
cd backend
.venv\Scripts\activate
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

## 8. API 设计

所有业务接口以 `/api` 开头。

音频静态路径不使用 `/api`：

```text
/audio/xxx.mp3
```

后端挂载：

```text
/audio -> data/audio
```

核心接口：

```text
GET    /api/health
GET    /api/dashboard/summary

GET    /api/knowledge-bases
POST   /api/knowledge-bases
PUT    /api/knowledge-bases/{id}
DELETE /api/knowledge-bases/{id}

POST   /api/materials
GET    /api/materials
GET    /api/materials/{id}
DELETE /api/materials/{id}

POST   /api/ai/extract-points
POST   /api/ai/generate-questions
POST   /api/ai/test-connection

GET    /api/knowledge-points
GET    /api/knowledge-points/{id}
DELETE /api/knowledge-points/{id}

GET    /api/questions
POST   /api/practice/submit

GET    /api/wrong-questions
DELETE /api/wrong-questions/{id}

POST   /api/tts/generate
GET    /api/audio
DELETE /api/audio/{id}

GET    /api/settings
PUT    /api/settings
POST   /api/settings/test-ai
POST   /api/settings/test-tts
```

返回格式建议：

```json
{
  "data": {},
  "message": "ok"
}
```

错误提示必须使用中文可读信息，避免只显示 `error`、`undefined`。

## 9. 数据库设计

数据库使用 SQLite。

数据库文件：

```text
data/app.db
```

禁止创建到：

```text
backend/data/app.db
```

必须通过 `backend/app/core/paths.py` 统一管理路径。

V1 核心表：

```text
knowledge_bases
materials
knowledge_points
questions
answer_records
wrong_questions
audio_files
app_settings
```

V1 可以使用：

```python
Base.metadata.create_all(bind=engine)
```

暂时不引入 Alembic，除非明确要求。

表关系：

```text
knowledge_bases 1 - n materials
knowledge_bases 1 - n knowledge_points
materials       1 - n knowledge_points
knowledge_points 1 - n questions
questions       1 - n answer_records
questions       1 - 1 wrong_questions
knowledge_points 1 - n audio_files
```


```text

```

```text

把真实 API Key 打印到日志
```




## 11. AI 设计

AI 能力必须由后端封装。

V1 只实现：

```text
AI 提取知识点
AI 生成题目
AI 连接测试
```

原则：

```text
使用 OpenAI-compatible Chat Completions
AI 输出尽量要求 JSON
后端必须容错解析 JSON
AI 失败时不要保存空数据或脏数据
AI 请求必须有 timeout
长耗时操作前端必须有 loading
```

必须支持解析：

```text
纯 JSON
Markdown JSON 代码块
前后有额外文本的 JSON
```

必须有：

```text
backend/app/utils/json_parser.py
```

知识点字段建议：

```text
title
summary
content
importance
tags
```

题目字段建议：

```text
knowledge_point_id
question_type
stem
options
answer
analysis
difficulty
```

题型：

```text
single_choice
true_false
```

## 12. TTS 设计

TTS 能力必须由后端封装。

V1 必须支持：

```text
mock provider
xiaomi provider 预留
```

如果真实小米 TTS 接口文档不明确，不写死接口地址。

正确做法：

```text
先实现 mock provider 打通流程
预留 XiaomiTTSProvider
从配置读取 XIAOMI_TTS_BASE_URL 和 XIAOMI_TTS_API_KEY
由用户后续填入真实接口参数
```

音频保存目录：

```text
data/audio/
```

音频命名：

```text
kp_{knowledge_point_id}_{timestamp}.mp3
```

数据库保存：

```text
file_path
file_url
status
error_message
```

状态：

```text
pending
success
failed
```

删除音频时：

```text
删除数据库记录
尝试删除本地音频文件
如果文件不存在，不报致命错误
```

## 13. Loading 与错误提示

以下操作必须有 loading：

```text
保存资料
保存并提取知识点
AI 生成题目
TTS 生成音频
提交答案
测试 AI 连接
测试 TTS 连接
删除数据
```

以下情况必须有中文错误提示：

```text
后端连接失败
AI API Key 未配置
AI 接口调用失败
AI JSON 解析失败
TTS API Key 未配置
TTS 生成失败
资料标题为空
资料正文为空
未选择知识库
没有题目
没有错题
没有音频
```
