# 05_AI与提示词设计

# AI 知识点学习与音频播报系统 - AI 与提示词设计

## 1. 文档目的

本文档用于定义本项目 V1 版本中 AI 相关能力的设计。

主要解决：

```text
AI 用来做什么？
后端如何调用 AI？
提示词怎么写？
AI 输出格式怎么约束？
AI 返回 JSON 失败怎么办？
如何降低幻觉和乱输出？
如何生成更适合考试复习的知识点和题目？
```

本项目的 AI 能力第一版只做两个核心功能：

```text
AI 提取知识点
AI 生成题目
```

后续再扩展：

```text
AI 生成复习讲稿
AI 生成音频播报稿
AI 总结错题
AI 生成每日复习计划
RAG 知识库问答
```

---

## 2. AI 功能定位

本项目中的 AI 不是普通聊天助手，而是一个学习资料处理器。

它主要负责：

```text
把原始资料整理成结构化知识点
把知识点转换成练习题
把学习内容改写成适合记忆和复习的形式
```

AI 不应该：

```text
随便扩展无关内容
编造资料中没有的信息
返回大段聊天式废话
输出无法解析的格式
忽略指定 JSON 格式
```

---

## 3. AI 接口设计原则

## 3.1 使用 OpenAI-compatible 接口

V1 建议统一使用 OpenAI-compatible Chat Completions 接口。

这样可以兼容：

```text
DeepSeek
OpenAI
硅基流动
通义千问兼容接口
其他兼容 OpenAI 格式的模型服务
```

配置项：

```env
AI_PROVIDER=deepseek
AI_API_KEY=your_api_key_here
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
AI_TEMPERATURE=0.3
```

---

## 3.2 后端统一封装

前端不能直接调用 AI。

所有 AI 调用必须由后端封装：

```text
frontend
↓
backend /api/ai/*
↓
ai_service.py
↓
AI provider
```

建议服务文件：

```text
backend/app/services/ai_service.py
```

核心方法：

```python
test_ai_connection()
extract_knowledge_points(material_title, material_content, knowledge_base_name)
generate_questions(knowledge_point, question_types, count)
```

---

## 3.3 Temperature 建议

因为本项目需要结构化输出，建议：

```text
AI_TEMPERATURE=0.2 ~ 0.4
```

默认：

```text
0.3
```

不要太高，否则容易输出发散、格式不稳定。

---

## 3.4 Token 与长度限制

V1 建议单次资料长度控制在：

```text
5000 ~ 10000 中文字以内
```

如果资料过长，前端或后端应提示：

```text
资料过长，建议分段导入。
```

后续版本再做自动分段。

---

## 4. AI 返回格式总原则

AI 必须尽量返回 JSON。

严禁返回：

```text
解释性开头
Markdown 标题
无关闲聊
代码块外的说明
```

推荐要求 AI 只返回 JSON：

```text
你必须只返回 JSON，不要返回任何解释、注释、Markdown 或额外文本。
```

但现实中模型可能会返回：

```text
```json
{...}
```

或者：

```text
以下是结果：
{...}
```

所以后端必须做容错解析。

---

## 5. JSON 解析策略

建议创建文件：

```text
backend/app/utils/json_parser.py
```

需要支持：

```text
1. 直接解析纯 JSON
2. 提取 ```json 代码块中的 JSON
3. 从文本中寻找第一个 { 到最后一个 } 的内容
4. 解析失败时抛出清晰错误
```

伪代码：

```python
def extract_json_from_text(text: str) -> dict:
    if not text:
        raise ValueError("AI 返回内容为空")

    # 1. 尝试直接 json.loads
    # 2. 尝试提取 ```json ... ```
    # 3. 尝试截取首个 { 到最后一个 }
    # 4. 仍失败则抛出异常
```

## 6. AI 提取知识点设计

## 6.1 功能说明

输入一段学习资料，让 AI 提取出多个结构化知识点。

适合资料：

```text
时政材料
公基知识
管理学概念
法律常识
经济常识
古文介绍
考试笔记
视频文稿
```

---

## 6.2 输入参数

后端传给 AI 的信息：

```text
知识库名称
资料标题
资料正文
提取要求
输出格式要求
```

示例：

```json
{
  "knowledge_base_name": "公共基础知识",
  "material_title": "政府社会职能",
  "material_content": "资料正文..."
}
```

---

## 6.3 输出字段

AI 必须输出：

```text
knowledge_points
```

每个知识点包含：

```text
title
summary
detail
exam_points
confusing_points
memory_tips
examples
importance
tags
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| title | string | 知识点标题 |
| summary | string | 一句话简短解释 |
| detail | string | 详细解释 |
| exam_points | array[string] | 高频考点 |
| confusing_points | array[string] | 易混点 |
| memory_tips | array[string] | 记忆方法 |
| examples | array[string] | 例子 |
| importance | string | low / medium / high |
| tags | array[string] | 标签 |

---

## 6.4 知识点提取 System Prompt

推荐 system prompt：

```text
你是一个考试备考资料整理助手，擅长把学习资料整理成结构化知识点。

你的任务：
1. 从用户提供的资料中提取适合复习和考试的知识点。
2. 输出必须严格符合指定 JSON 格式。
3. 不要编造资料中没有的信息。
4. 不要输出 Markdown。
5. 不要输出解释性文字。
6. 不要把无关内容扩展成知识点。
7. 知识点要适合中国考试类复习场景，尤其是公共基础知识、事业单位、三支一扶、时政、管理学、法律、经济、语文古文等。
8. 如果资料中有明显的概念、分类、区别、意义、作用、原则、特点、例子，要优先提取。
9. 每个知识点要简洁、准确、可复习。
10. 高频考点和易混点要尽量站在考试角度总结。

你必须只返回 JSON，不要返回任何额外文本。
```

---

## 6.5 知识点提取 User Prompt 模板

```text
请根据下面的学习资料，提取结构化知识点。

知识库名称：
{{knowledge_base_name}}

资料标题：
{{material_title}}

资料正文：
{{material_content}}

提取要求：
1. 提取 3 到 10 个核心知识点。
2. 如果资料较短，可以少于 3 个。
3. 每个知识点必须来自资料内容，不要编造。
4. 每个知识点要包含简短解释、详细解释、高频考点、易混点、记忆方法和示例。
5. 高频考点要适合考试复习。
6. 易混点要指出容易和哪些概念混淆。
7. 记忆方法要尽量短，方便背诵。
8. importance 只能是 low、medium、high。
9. tags 使用 1 到 5 个中文标签。
10. 必须只返回 JSON，不要返回 Markdown。

返回 JSON 格式必须如下：

{
  "knowledge_points": [
    {
      "title": "知识点标题",
      "summary": "一句话解释",
      "detail": "详细解释",
      "exam_points": ["考点1", "考点2"],

      "memory_tips": ["记忆方法1"],
      "examples": ["例子1", "例子2"],
      "importance": "high",
      "tags": ["标签1", "标签2"]
    }
  ]
}
```

## 6.7 知识点提取质量要求

好的知识点应该满足：

```text
标题清楚
解释准确
考点具体
易混点有价值
记忆方法短
示例贴近考试
不要废话
不要过度扩展
```

不好的知识点：

```text
标题太大：政府

考点无意义：要认真学习政府职能
易混点为空
记忆方法太长
编造资料中没有的信息
```

## 7. AI 生成题目设计

## 7.1 功能说明

根据一个知识点，生成适合复习的练习题。

V1 支持：

```text
单选题
判断题
```

后续扩展：

```text
多选题
简答题
材料分析题
```

---

## 7.2 输入参数

后端传给 AI：

```text
知识点标题
简短解释
详细解释
高频考点
易混点
记忆方法
示例
题型
题目数量
```

示例：

```json
{
  "knowledge_point_id": 1,
  "title": "政府社会职能",
  "summary": "...",
  "detail": "...",
  "exam_points": ["..."],
  "confusing_points": ["..."],
  "question_types": ["single_choice", "true_false"],
  "count": 5
}
```

---

## 7.3 输出字段

AI 必须返回：

```text
questions
```

每道题包含：

```text
question_type
stem
options
answer
analysis
difficulty
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| question_type | string | single_choice / true_false |
| stem | string | 题干 |
| options | array[object] | 选项 |
| answer | string | 正确答案 |
| analysis | string | 解析 |
| difficulty | string | easy / medium / hard |

---

## 7.4 题型规则

### 7.4.1 单选题 single_choice

要求：

```text
必须有 4 个选项
选项 key 必须是 A、B、C、D
只能有一个正确答案
answer 必须是 A/B/C/D 之一
干扰项要合理
解析要说明为什么正确，最好也说明其他选项为什么不合适
```

选项格式：

```json
[
  { "key": "A", "text": "选项 A" },
  { "key": "B", "text": "选项 B" },
  { "key": "C", "text": "选项 C" },
  { "key": "D", "text": "选项 D" }
]
```

---

### 7.4.2 判断题 true_false

要求：

```text
options 固定为 true / false
answer 必须是 true 或 false
题干必须是一个明确可判断的陈述句
解析必须说明判断依据
```

选项格式：

```json
[
  { "key": "true", "text": "正确" },
  { "key": "false", "text": "错误" }
]
```

---

## 7.5 AI 生成题目 System Prompt

```text
你是一个考试题目生成助手，擅长根据知识点生成适合复习的练习题。

你的任务：
1. 根据用户提供的知识点生成题目。
2. 题目必须围绕知识点本身，不要偏题。
3. 不要编造与知识点无关的信息。
4. 单选题必须有 A、B、C、D 四个选项，且只有一个正确答案。
5. 判断题必须使用 true / false 作为答案。
6. 干扰项要合理，不能明显离谱。
7. 解析要清楚说明正确原因。
8. 输出必须严格符合指定 JSON 格式。
9. 不要输出 Markdown。
10. 不要输出任何解释性文字。
11. difficulty 只能是 easy、medium、hard。
12. question_type 只能使用用户指定的题型。

你必须只返回 JSON，不要返回任何额外文本。
```

## 9. AI 服务返回校验

AI 返回 JSON 后，后端不能直接保存，必须做基础校验。

---

## 9.1 知识点校验

必须检查：

```text
knowledge_points 是否存在
knowledge_points 是否为数组
每个知识点是否有 title
importance 是否在 low / medium / high 内
数组字段如果不是数组，尝试转为空数组
```

如果缺少 summary / detail，可以允许为空。

如果没有 title，则跳过该知识点。

---

## 9.2 题目校验

必须检查：

```text
questions 是否存在
questions 是否为数组
question_type 是否合法
stem 是否存在
answer 是否存在
difficulty 是否合法
single_choice 是否有 A/B/C/D 四个选项
single_choice answer 是否为 A/B/C/D
true_false answer 是否为 true/false
```

不合格的题目应跳过，不要保存。

如果全部不合格，返回错误：

```text
AI 返回的题目格式不符合要求，请重试。
```

---

## 10. AI 失败重试策略

V1 不需要复杂重试，但可以做简单处理。

推荐：

```text
AI 请求失败：不自动重试，直接提示用户
AI JSON 解析失败：可以自动重试 1 次
AI 返回空结果：提示用户重试
```

后续可加：

```text
重新格式化 JSON 的修复请求
```

---

## 11. AI JSON 修复 Prompt

如果 AI 返回了格式不正确的内容，可以让模型再修复一次。

System Prompt：

```text
你是一个 JSON 格式修复助手。你只负责把输入内容修复为合法 JSON，不添加任何新信息。
```

User Prompt：

```text
下面内容本应是 JSON，但格式可能不正确。请在不改变原始含义、不添加新信息的前提下，修复为合法 JSON。

原始内容：
{{raw_ai_output}}

要求：
1. 只返回合法 JSON。
2. 不要返回 Markdown。
3. 不要解释。
4. 不要添加原始内容中没有的信息。
```

V1 可以先不实现，作为后续优化。

---

## 12. AI 输出安全边界

本项目主要处理学习资料，但仍应设置边界。

AI 不应：

```text
生成违法内容
生成考试作弊答案
伪造官方文件
生成攻击性内容
泄露 API Key
输出用户隐私信息
```

如果资料中包含敏感个人信息，AI 提取时应尽量避免不必要地保留。

## 19. AI 成本控制

V1 应避免无意义重复调用。

前端应该：

```text
AI 操作按钮 loading 时禁用
AI 生成前弹出确认或显示预计操作
不要页面刷新自动调用 AI
```

后端应该：

```text
限制 count 最大值
限制资料长度
记录 AI 原始返回，方便排错
```

推荐题目生成数量限制：

```text
1 到 20 道
```

推荐知识点提取数量：

```text
3 到 10 个
```

## 20. V1 AI 验收标准

完成 AI 部分后，应满足：

```text
1. 设置页可以配置 AI API Key、Base URL、Model。
2. 可以测试 AI 连接。
3. 可以对一段资料提取知识点。
4. AI 返回的知识点能保存到数据库。
5. 知识点字段包含标题、解释、考点、易混点、记忆方法。
6. 可以根据知识点生成单选题。
7. 可以根据知识点生成判断题。
8. 题目能保存到数据库。
9. AI 返回 Markdown JSON 代码块时，后端也能解析。
10. AI 返回格式错误时，前端有明确提示。
11. API Key 不会写死到代码里。
12. AI 请求期间前端有 loading 状态。
```
