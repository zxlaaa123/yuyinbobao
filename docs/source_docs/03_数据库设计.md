# 03_数据库设计

# AI 知识点学习与音频播报系统 - 数据库设计

## 1. 文档目的

本文档用于定义本项目 V1 版本的 SQLite 数据库结构。

它主要回答：

```text
需要哪些表？
每张表保存什么？
字段类型是什么？
表之间怎么关联？
哪些字段是必填？
哪些字段是枚举值？
后端 SQLAlchemy 模型应该怎么设计？
```

V1 使用 SQLite，本地数据库文件保存到：

```text
data/app.db
```

---

## 2. 数据库设计原则

V1 数据库设计遵循以下原则：

```text
简单优先
够用即可
字段清晰
方便后续扩展
避免过度设计
```

V1 暂不考虑：

```text
多用户
权限系统
云同步
复杂审计日志
复杂版本控制
分布式数据库
```

---

## 3. 数据表总览

V1 核心数据表：

```text
knowledge_bases      知识库表
materials            学习资料表
knowledge_points     知识点表
questions            题目表
answer_records       答题记录表
wrong_questions      错题表
audio_files          音频文件表
app_settings         系统设置表
```

后续版本可增加：

```text
review_tasks         复习任务表
tags                 标签表
file_imports         文件导入记录表
rag_documents        RAG 文档表
rag_chunks           RAG 分块表
```

---

## 4. 表关系总览

核心关系：

```text
knowledge_bases 1 - N materials
knowledge_bases 1 - N knowledge_points
materials 1 - N knowledge_points
knowledge_points 1 - N questions
questions 1 - N answer_records
questions 1 - 1 wrong_questions
knowledge_points 1 - N audio_files
```

简单理解：

```text
一个知识库下面有多篇资料
一篇资料可以提取多个知识点
一个知识点可以生成多道题
一道题可以有多次答题记录
一道题如果答错，会进入错题表
一个知识点可以生成多个音频文件
```

# 5. 表结构详情

## 5.1 knowledge_bases 知识库表

### 5.1.1 作用

保存用户创建的知识库分类。

示例：

```text
三支一扶
公共基础知识
时政
管理学
法律常识
经济常识
古文
```

---

### 5.1.2 字段设计

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| id | Integer | 是 | 自增 | 主键 |
| name | String(100) | 是 | 无 | 知识库名称 |
| description | Text | 否 | 空 | 知识库描述 |
| sort_order | Integer | 否 | 0 | 排序值 |
| created_at | DateTime | 是 | 当前时间 | 创建时间 |
| updated_at | DateTime | 是 | 当前时间 | 更新时间 |

---

### 5.1.3 约束建议

```text
name 不为空
name 建议唯一
```

SQLite 可设置：

```text
UNIQUE(name)
```

如果担心用户想创建重名知识库，也可以不做唯一约束。

V1 推荐：

```text
name 唯一
```

---

### 5.1.4 SQLAlchemy 模型建议

```python
class KnowledgeBase(Base):
    __tablename__ = "knowledge_bases"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(100), nullable=False, unique=True, index=True)
    description = Column(Text, nullable=True)
    sort_order = Column(Integer, default=0)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

## 5.2 materials 学习资料表

### 5.2.1 作用

保存用户导入或粘贴的原始学习资料。

V1 只支持粘贴文本。

后续可扩展：

```text
PDF
Word
Markdown
TXT
网页链接
```

---

### 5.2.2 字段设计

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| id | Integer | 是 | 自增 | 主键 |
| knowledge_base_id | Integer | 是 | 无 | 所属知识库 ID |
| title | String(200) | 是 | 无 | 资料标题 |
| content | Text | 是 | 无 | 资料正文 |
| source | String(300) | 否 | 空 | 资料来源 |
| note | Text | 否 | 空 | 备注 |
| material_type | String(50) | 是 | text | 资料类型 |
| file_path | String(500) | 否 | 空 | 原始文件路径，V1 可为空 |
| content_length | Integer | 否 | 0 | 正文字数 |
| extracted_count | Integer | 否 | 0 | 已提取知识点数量 |
| created_at | DateTime | 是 | 当前时间 | 创建时间 |
| updated_at | DateTime | 是 | 当前时间 | 更新时间 |

---

### 5.2.3 字段说明

`material_type` 可选值：

```text
text
pdf
word
markdown
txt
web
```

V1 只使用：

```text
text
```

`content_length` 保存资料正文长度，方便后续统计和限制 AI 输入长度。

---

### 5.2.4 外键关系

```text
knowledge_base_id → knowledge_bases.id
```

删除知识库时，V1 可以级联删除资料。

---

### 5.2.5 SQLAlchemy 模型建议

```python
class Material(Base):
    __tablename__ = "materials"

    id = Column(Integer, primary_key=True, index=True)
    knowledge_base_id = Column(Integer, ForeignKey("knowledge_bases.id"), nullable=False, index=True)
    title = Column(String(200), nullable=False, index=True)
    content = Column(Text, nullable=False)
    source = Column(String(300), nullable=True)
    note = Column(Text, nullable=True)
    material_type = Column(String(50), default="text", nullable=False)
    file_path = Column(String(500), nullable=True)
    content_length = Column(Integer, default=0)
    extracted_count = Column(Integer, default=0)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

## 5.3 knowledge_points 知识点表

### 5.3.1 作用

保存 AI 从资料中提取出来的结构化知识点。

这是项目的核心表之一。

---

### 5.3.2 字段设计

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| id | Integer | 是 | 自增 | 主键 |
| knowledge_base_id | Integer | 是 | 无 | 所属知识库 ID |
| material_id | Integer | 否 | 空 | 来源资料 ID |
| title | String(200) | 是 | 无 | 知识点标题 |
| summary | Text | 否 | 空 | 简短解释 |
| detail | Text | 否 | 空 | 详细解释 |
| exam_points | Text | 否 | 空 | 高频考点，JSON 字符串 |
| confusing_points | Text | 否 | 空 | 易混点，JSON 字符串 |
| memory_tips | Text | 否 | 空 | 记忆方法，JSON 字符串 |
| examples | Text | 否 | 空 | 示例，JSON 字符串 |
| importance | String(20) | 是 | medium | 重要程度 |
| tags | Text | 否 | 空 | 标签，JSON 字符串 |
| ai_raw_response | Text | 否 | 空 | AI 原始返回，便于排错 |
| created_at | DateTime | 是 | 当前时间 | 创建时间 |
| updated_at | DateTime | 是 | 当前时间 | 更新时间 |

---

### 5.3.3 字段说明

以下字段建议以 JSON 字符串保存：

```text
exam_points
confusing_points
memory_tips
examples
tags
```

例如：

```json
["社会保障属于政府社会职能", "生态环境保护属于政府社会职能"]
```

V1 使用 Text 保存 JSON 字符串，避免引入复杂 JSON 字段兼容问题。

---

### 5.3.4 importance 可选值

```text
low
medium
high
```

默认：

```text
medium
```

含义：

```text
low       低频或补充知识点
medium    普通知识点
high      高频考点或重点知识点
```

---

### 5.3.5 外键关系

```text
knowledge_base_id → knowledge_bases.id
material_id → materials.id
```

`material_id` 可为空，方便后续手动创建知识点。

---

### 5.3.6 SQLAlchemy 模型建议

```python
class KnowledgePoint(Base):
    __tablename__ = "knowledge_points"

    id = Column(Integer, primary_key=True, index=True)
    knowledge_base_id = Column(Integer, ForeignKey("knowledge_bases.id"), nullable=False, index=True)
    material_id = Column(Integer, ForeignKey("materials.id"), nullable=True, index=True)
    title = Column(String(200), nullable=False, index=True)
    summary = Column(Text, nullable=True)
    detail = Column(Text, nullable=True)
    exam_points = Column(Text, nullable=True)
    confusing_points = Column(Text, nullable=True)
    memory_tips = Column(Text, nullable=True)
    examples = Column(Text, nullable=True)
    importance = Column(String(20), default="medium", nullable=False, index=True)

    ai_raw_response = Column(Text, nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
```

## 5.6 wrong_questions 错题表

### 5.6.1 作用

保存用户答错的题目。

同一道题多次答错时，不重复新增记录，而是增加 `wrong_count`。

---

### 5.6.2 字段设计

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| id | Integer | 是 | 自增 | 主键 |
| question_id | Integer | 是 | 无 | 题目 ID |
| wrong_count | Integer | 是 | 1 | 错误次数 |
| last_wrong_answer | String(100) | 否 | 空 | 最近一次错误答案 |
| last_wrong_at | DateTime | 是 | 当前时间 | 最近一次答错时间 |
| is_mastered | Boolean | 是 | false | 是否已掌握 |
| created_at | DateTime | 是 | 当前时间 | 创建时间 |
| updated_at | DateTime | 是 | 当前时间 | 更新时间 |

---

### 5.6.3 约束建议

```text
question_id 唯一
```

确保一道题只在错题本中出现一次。

---

### 5.6.4 业务规则

当用户答错时：

```text
如果 question_id 不存在于 wrong_questions：
    新增错题记录
    wrong_count = 1
    is_mastered = false

如果 question_id 已存在：
    wrong_count + 1
    更新 last_wrong_answer
    更新 last_wrong_at
    is_mastered = false
```

当用户点击“标记已掌握”：

```text
is_mastered = true
```

---

### 5.6.5 SQLAlchemy 模型建议

```python
class WrongQuestion(Base):
    __tablename__ = "wrong_questions"

    id = Column(Integer, primary_key=True, index=True)
    question_id = Column(Integer, ForeignKey("questions.id"), nullable=False, unique=True, index=True)
    wrong_count = Column(Integer, default=1, nullable=False)
    last_wrong_answer = Column(String(100), nullable=True)
    last_wrong_at = Column(DateTime, default=datetime.utcnow)
    is_mastered = Column(Boolean, default=False, nullable=False, index=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

## 9. 删除规则

V1 删除规则建议：

### 9.1 删除知识库

删除知识库时级联删除：

```text
materials
knowledge_points
questions
answer_records
wrong_questions
audio_files
```

同时可以删除对应音频文件。

V1 如果不想实现复杂级联，可以先禁止删除非空知识库，提示：

```text
该知识库下还有资料或知识点，请先清空后再删除。
```

推荐 V1 采用更安全方案：

```text
非空知识库禁止删除
```

---

### 9.2 删除学习资料

删除资料时：

```text
可以保留已提取知识点
```

或者：

```text
级联删除由该资料提取的知识点
```

V1 推荐：

```text
删除资料时不自动删除知识点
```

避免误删学习成果。

---

### 9.3 删除知识点

删除知识点时，应删除：

```text
关联题目
关联错题
关联答题记录
关联音频记录
```

音频实际文件也应尝试删除。

---

### 9.4 删除音频记录

删除音频记录时：

```text
删除 audio_files 表记录
尝试删除 data/audio 下的实际 mp3 文件
```

如果文件不存在，不应报致命错误。

## 14. 数据库文件位置注意事项

数据库文件必须始终创建在项目根目录下的：

```text
data/app.db
```

不能因为后端启动目录不同，创建到：

```text
backend/data/app.db
```

所以后端必须用统一路径计算。

建议：

```text
backend/app/core/paths.py
```

负责确定项目根目录和 data 目录。

## 15. V1 数据库验收标准

完成数据库部分后，应满足：

```text
1. 启动后端时自动创建 data/ 目录。
2. 启动后端时自动创建 data/app.db。
3. 启动后端时自动创建所有 V1 数据表。
4. 可以成功创建知识库。
5. 可以成功保存学习资料。
6. 可以成功保存知识点。
7. 可以成功保存题目。
8. 提交答案后可以保存答题记录。
9. 答错后可以保存或更新错题。
10. 生成音频后可以保存 audio_files 记录。
11. 设置页可以读取和保存 app_settings。
12. 关闭后端重新启动，数据仍然存在。
```
