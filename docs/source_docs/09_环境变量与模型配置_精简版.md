# 09_环境变量与模型配置_精简版

# AI 知识点学习与音频播报系统 - 环境变量与模型配置

## 1. 文档目的

本文档定义项目 V1 的环境变量、模型配置和密钥管理方式。

重点解决：

```text
.env 怎么写
API Key 放哪里
AI / TTS 如何配置
设置页和 .env 谁优先
如何避免密钥提交到 Git
```

---

## 2. 配置原则

```text
敏感信息不写死
真实 API Key 不提交 Git
.env.example 可以提交
.env 不可以提交
前端不保存 AI / TTS Key
外部 AI / TTS 调用必须走后端
```

---

## 3. 配置来源

V1 推荐两个配置来源：

```text
.env
app_settings 数据库表
```

读取优先级：

```text
app_settings 数据库 > .env > 默认值
```

含义：

```text
设置页保存过的配置优先
没有设置页配置时读取 .env
都没有则使用默认值
```

---


## 5. .env.example 推荐内容

```env
# 应用基础配置
APP_NAME=AI Study Cast
APP_ENV=development
APP_VERSION=0.1.0

BACKEND_HOST=127.0.0.1
BACKEND_PORT=8000
FRONTEND_URL=http://localhost:5173

# 数据库
DATABASE_URL=sqlite:///data/app.db

# AI 模型
AI_PROVIDER=deepseek
AI_API_KEY=
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
AI_TEMPERATURE=0.3
AI_TIMEOUT=120

# TTS 音频
TTS_PROVIDER=mock
XIAOMI_TTS_API_KEY=
XIAOMI_TTS_BASE_URL=
XIAOMI_TTS_VOICE=default
XIAOMI_TTS_FORMAT=mp3
XIAOMI_TTS_SPEED=1.0
XIAOMI_TTS_TIMEOUT=120

# 本地目录
DATA_DIR=data
UPLOAD_DIR=data/uploads
AUDIO_DIR=data/audio
VECTOR_STORE_DIR=data/vector_store

# 前端
VITE_API_BASE_URL=http://localhost:8000
```


```text

减少跑题

```

## 10. AI URL 拼接规则

不同服务的 base URL 可能是：

```text
https://api.openai.com
https://api.openai.com/v1
```

建议后端统一处理，避免重复 `/v1`：

```text
如果 base_url 以 /v1 结尾：
  使用 {base_url}/chat/completions

否则：
  使用 {base_url}/v1/chat/completions
```

## 11. TTS 配置

### 配置项

```text
TTS_PROVIDER
XIAOMI_TTS_API_KEY
XIAOMI_TTS_BASE_URL
XIAOMI_TTS_VOICE
XIAOMI_TTS_FORMAT
XIAOMI_TTS_SPEED
XIAOMI_TTS_TIMEOUT
```

### 默认

```env
TTS_PROVIDER=mock
```

原因：

```text
没有真实小米 TTS Key 时，也能先打通音频生成流程
```

### 小米 TTS

```env
TTS_PROVIDER=xiaomi
XIAOMI_TTS_API_KEY=你的_key
XIAOMI_TTS_BASE_URL=按官方文档填写
XIAOMI_TTS_VOICE=default
XIAOMI_TTS_FORMAT=mp3
XIAOMI_TTS_SPEED=1.0
XIAOMI_TTS_TIMEOUT=120
```

注意：

```text
小米 TTS 的具体接口地址、鉴权方式、参数名，以官方文档为准
不要在代码中瞎写死
```

## 13. 前端环境变量

Vite 前端变量必须以：

```text
VITE_
```

开头。

允许：

```env
VITE_API_BASE_URL=http://localhost:8000
```

禁止：

```env
VITE_AI_API_KEY=xxx
VITE_XIAOMI_TTS_API_KEY=xxx
```

原因：

```text
前端环境变量会暴露给浏览器
```

## 16. 敏感字段规则

### 显示时脱敏

示例：

```text
sk-1234567890abcd → sk-1****abcd
abcd1234 → ****
```

### 保存时规则

```text
用户填写新 Key：覆盖旧值
用户留空 Key：保留旧值
```

除非提供明确按钮：

```text
清除 API Key
```

### 日志规则

禁止打印完整 Key。

允许：

```text
AI API Key 已配置
AI API Key = sk-1****abcd
```

## 19. 配置校验

调用 AI 前检查：

```text
AI_API_KEY
AI_BASE_URL
AI_MODEL
```

缺失时返回：

```json
{
  "detail": "AI API Key 未配置，请先到设置页配置"
}
```

调用 TTS 前检查：

```text
TTS_PROVIDER
如果 provider=xiaomi，检查 XIAOMI_TTS_API_KEY 和 XIAOMI_TTS_BASE_URL
如果 provider=mock，不需要真实 Key
```

## 21. Windows 使用流程

首次使用：

```text
1. 复制 .env.example 为 .env
2. 填写 AI_API_KEY
3. 根据需要填写 TTS 配置
4. 运行 scripts/init_backend.bat
5. 运行 scripts/init_frontend.bat
6. 双击 start.bat
```

日常使用：

```text
双击 start.bat
浏览器打开 http://localhost:5173
```

## 22. 配置模块验收标准

```text
.env.example 存在
.env 被 .gitignore 忽略
后端能从 .env 读取配置
后端能从 app_settings 读取配置
app_settings 优先级高于 .env
设置页可保存 AI 配置
设置页可保存 TTS 配置
API Key 显示时脱敏
敏感字段留空不覆盖旧值
测试 AI 连接有明确提示
测试 TTS 连接有明确提示
日志不打印完整 API Key
```

---

## 23. 总结

配置模块核心原则：

```text
配置和代码分离
密钥不进 Git
后端统一管理
前端只调用后端
设置页方便修改
.env 作为本地基础配置
```
